# 알고리즘 *'실전 5단계 루틴'*

---

### **1. 🔍 요구사항 분석 및 유형 파악 (Upgrade!)**
* **문제 유형:** (예: BFS 최단거리, 2차원 배열 시뮬레이션, 완전 탐색 등)
* **유형 파악 키워드:** 이 문제를 해당 유형으로 판단하게 만든 결정적인 단서나 문장 (예: "최단 시간을 구하시오", "N이 10 이하이다", "상하좌우로 이동한다" 등)
* **핵심 목표:** 문제에서 요구하는 진짜 의도와 결과물
* **입력 범위 및 시간 복잡도:** $N$의 크기를 확인하고 목표 시간 복잡도 역산
* **주의할 엣지 케이스:** 극단적인 입력값(최소/최대)이나 예외 처리

### **2. 🧠 논리적 설계 (주석 뼈대)**
* [풀었을 경우] 보내주신 주석을 분석하여 논리적 흐름 피드백
* [못 풀었을 경우] 최적의 해결 과정을 1~3단계 한글 주석으로 구조화

### **3. 💻 코드 구현 (Pythonic)**
* [풀었을 경우] 작성하신 코드의 아쉬운 점이나 더 짧고 빠르게 짤 수 있는 파이썬 내장 함수 활용법 리뷰
* [못 풀었을 경우] 설계한 뼈대를 바탕으로 작성한 깔끔하고 직관적인 코드 제공

### **4. 🛡️ 엣지 케이스 점검**
* 1단계에서 찾은 엣지 케이스 방어 여부 및 런타임 에러 위험 구간 점검

### **5. 🏆 최고 수준 최적화 (Review & Optimization)**
* 실행 시간과 메모리 사용량을 극한으로 줄인 **가장 최적화된 파이썬 정답 코드**와 그 핵심 원리 제공
