### 1. 🔍 요구사항 분석 및 유형 파악 (Upgrade!)

* **문제 유형:** 동적 계획법 (Dynamic Programming, DP), 수학
* **유형 파악 키워드:** "피보나치 수", "$n$은 45보다 작거나 같은 자연수"
    * 이전의 두 항을 더해 다음 항을 만드는 점화식 구조, 그리고 단순 재귀 연산 시 기하급수적으로 늘어나는 시간 복잡도($O(2^N)$)를 피하기 위해 이미 구한 값을 저장하고 재활용(메모이제이션)해야 한다는
      결정적 단서입니다.
* **핵심 목표:** $F_n = F_{n-1} + F_{n-2}$ 점화식을 중복 계산 없이 $n$번째 항까지 빠르게 도출하기.
* **입력 범위 및 시간 복잡도:** $N \le 45$. DP 배열을 이용하면 $O(N)$ 시간 복잡도를 가지므로, 45번의 연산만으로 1초 제한을 아주 여유롭게 통과합니다.
* **주의할 엣지 케이스:** $N=0$이 들어올 경우와 $N=1$이 들어올 때 인덱스 에러가 나지 않게 초기값을 잘 방어해야 합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

작성하신 정석적인 바텀업 DP 로직의 뼈대입니다.

```markdown
1. 계산된 값을 저장할 DP 배열(리스트)을 생성한다. (N의 최대 크기인 45까지 담을 수 있게 46 크기로)
2. 피보나치 수열의 초기값인 F(0) = 0과 F(1) = 1을 배열에 세팅한다.
3. N을 입력받는다.
4. 2부터 N까지 반복문을 돌며, 점화식(현재 값 = 직전 값 + 그 직전 값)을 이용해 배열을 채워나간다.
5. 최종적으로 완성된 배열의 N번째 인덱스 값을 출력한다.
```

### 3. 💻 코드 구현 리뷰 (파이써닉 피드백)

작성하신 코드는 정답으로 전혀 손색이 없는 훌륭한 DP 코드입니다. `lst[0]`과 `lst[1]`을 먼저 세팅하고 배열의 크기를 46으로 고정해두어 어떤 값이 들어오든 런타임 에러를 방어한 점이 아주 좋습니다.

**[아쉬운 점 한 가지]**
`for i in range(2, 46):`
입력받은 `n`이 5라면 5까지만 계산해도 충분한데, 무조건 45번째 피보나치 수까지 끝까지 계산하고 있습니다. `range(2, n + 1)`로 수정하면 불필요한 연산을 줄일 수 있습니다. (물론 45번 연산
자체가 너무 작아서 시간 차이는 거의 없습니다!)

### 4. 🛡️ 엣지 케이스 점검

* **$N=0$ 또는 $N=1$인 경우:** 반복문 `for i in range(2, 46)`에 진입하더라도 어차피 리스트의 0번, 1번 인덱스는 처음에 올바르게 세팅되어 있으므로 정답이 무사히 출력됩니다.
* **인덱스 에러 방지:** 빈 배열 `lst = []`에 `append`를 하는 대신, 처음부터 `[0] * 46`으로 고정 크기를 할당해 둔 덕분에 인덱스 참조 에러를 원천 차단했습니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

현재 코드는 크기가 46인 배열을 사용하므로 공간 복잡도가 $O(N)$입니다.
하지만 피보나치 수열은 "가장 최근의 두 숫자"만 기억하면 다음 숫자를 만들 수 있으므로, **배열 전체를 유지할 필요 없이 변수 2개만 갱신하며 나아가는 $O(1)$ 메모리 최적화**가 가능합니다. 파이썬의 '다중
할당' 문법이 이를 마법처럼 짧게 만들어줍니다.

```python
import sys

input = sys.stdin.readline

n = int(input())

# 초기값 세팅 (a는 F(0), b는 F(1) 역할)
a, b = 0, 1

# n번 반복하며 갱신
for _ in range(n):
    # 파이썬 다중 할당: 다음 턴의 a는 현재 b가 되고, 다음 턴의 b는 현재 a+b가 된다
    a, b = b, a + b

# n번 갱신 후 a에 들어있는 값이 n번째 피보나치 수가 됨
print(a)
```

💡 **최적화 핵심 원리:**

이 기법을 알고리즘에서는 **'슬라이딩 윈도우 DP (Sliding Window DP)'** 또는 **'공간 최적화'** 라고 부릅니다.
파이썬의 `a, b = b, a + b` 다중 할당은 내부적으로 임시 값들을 동시에 바꿔치기해주기 때문에, 별도의 `temp` 변수나 배열(List) 메모리를 전혀 쓰지 않고도 피보나치 수를 완벽하게 구해냅니다. 이
3줄짜리 테크닉은 나중에 코딩테스트에서 아주 자주 써먹게 되실 겁니다!
