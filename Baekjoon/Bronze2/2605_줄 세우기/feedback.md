### 1. 🔍 요구사항 분석 및 유형 파악

* **문제 유형:** 자료구조 (Data Structure), 구현 및 시뮬레이션 (Simulation)
* **유형 파악 키워드:** "번호를 뽑아 자신의 자리를 찾아간다", "줄을 선다"
    * 특정 위치에 데이터를 밀어 넣는 **삽입(Insert)** 작업이 지속해서 일어나는 전형적인 배열 조작 유형입니다.
* **핵심 목표:** 학생들이 뽑은 숫자에 맞추어 결과 리스트의 특정 인덱스에 학생 번호를 끼워 넣기.
* **입력 범위 및 시간 복잡도:** 학생 수 $N$은 최대 100입니다. 데이터 크기가 매우 작으므로, 삽입 연산을 100번 반복하는 알고리즘으로도 시간 제한 내에 아주 여유롭게 통과합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

작성해주신 로직의 직관적인 흐름을 실전 뼈대로 정리하면 다음과 같습니다.

```markdown
1. 학생 수 N을 입력받는다.
2. 학생들이 뽑은 번호 배열을 입력받는다.
3. 최종 줄을 세울 빈 결과 리스트(result)를 만든다.
4. 1번 학생부터 N번 학생까지 순서대로 나아가며,
5. 해당 학생이 뽑은 번호를 확인하고, 결과 리스트의 뒤에서부터 '뽑은 번호'만큼 앞으로 간 위치에 학생 번호를 삽입한다.
6. 최종 완성된 결과 리스트를 공백으로 구분하여 출력한다.
```

### 3. 💻 코드 구현 리뷰

슬라이싱을 통한 삽입 논리는 완벽하게 동작합니다. 다만, 메모리와 가독성 측면에서 두 가지 파이써닉한 대안을 제시해 드립니다.

* **리스트 재생성 방지:** `result = result[:X] + [student] + result[X:]` 구문은 리스트 간의 덧셈(`+`) 연산을 수행할 때마다 메모리에 완전히 새로운 리스트 객체를 만들어
  냅니다. 데이터가 작을 땐 상관없지만, 커지면 효율이 떨어집니다. 파이썬 리스트에는 이 역할을 그 자리에서 바로 수행해 주는 `.insert()` 메서드가 있습니다.
* **출력 포맷팅:** `.join(list(map(str, result)))`도 훌륭한 정석이지만, 코딩테스트에서는 타자 시간을 1초라도 줄이는 것이 좋습니다. 파이썬의 **언패킹(Unpacking)** 연산자를
  쓰면 훨씬 간단해집니다.

### 4. 🛡️ 엣지 케이스 점검

* **첫 번째 학생의 처리:** 문제 조건상 첫 번째 학생은 무조건 0을 뽑습니다. `len(result) - 0`은 0이 되므로, 인덱스 0 위치에 제일 먼저 깔끔하게 삽입됩니다.
* **제일 앞으로 가는 경우:** 리스트에 이미 3명이 있는데 3번을 뽑았다면, `3 - 3 = 0`이 되어 제일 앞자리인 0번 인덱스에 오차 없이 안착합니다. 예외 처리 방어가 아주 단단합니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

리뷰 내용을 바탕으로 파이썬의 내장 함수 `.insert()`와 언패킹 연산자 `*`를 적용하여 코드를 극한으로 줄인 최적화 버전입니다.

```python
import sys

input = sys.stdin.readline

N = int(input())
orders = list(map(int, input().split()))
result = []

for i in range(N):
    # i는 0부터 시작하므로 학생 번호는 i + 1
    # 삽입할 인덱스: (현재 줄 서 있는 사람 수) - (뽑은 번호)
    insert_idx = len(result) - orders[i]

    # [최적화 1] 슬라이싱 대신 insert() 메서드 활용
    result.insert(insert_idx, i + 1)

# [최적화 2] join과 map을 대체하는 언패킹 출력
print(*result)
```

💡 **최적화 핵심 원리:**

1. **`list.insert(index, element)`**: 지정한 `index` 위치에 `element`를 삽입합니다. 리스트 슬라이싱 덧셈 연산은 매번 새로운 리스트를 메모리에 할당하지만, `insert`
   는 기존 리스트의 내부 요소를 한 칸씩 뒤로 밀어내어 공간을 확보한 뒤 값을 넣으므로 메모리 활용 면에서 훨씬 효율적입니다.
2. **언패킹 연산자 `*` (Asterisk)**: `print` 함수 안에서 리스트 앞에 `*`를 붙이면, 리스트를 감싸고 있는 대괄호를 벗겨내고 안의 요소들을 공백(스페이스바) 한 칸을 기준으로 나열해
   줍니다. 배열의 요소를 한 줄로 출력해야 하는 문제에서 압도적으로 타자 수를 줄여주는 필수 스니펫(Snippet)입니다.
