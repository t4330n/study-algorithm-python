### 1. 🔍 요구사항 분석 및 유형 파악

* **문제 유형:** 수학 (Mathematics), 완전 탐색 (Brute-Force)
* **유형 파악 키워드:** "회전시켜서 같아지는 직사각형은 같은 직사각형이다" (중복 제거의 필요성), "넓이가 n 이하인 직사각형"
* **핵심 목표:** 1부터 $n$까지의 크기를 가질 수 있는 직사각형 중에서, 고유한 (가로, 세로) 쌍의 개수를 구하기.
* **입력 범위 및 시간 복잡도:** $n$은 최대 10,000입니다. 넓이가 $n$을 넘는 순간 `break`를 걸어주셨기 때문에 전체 반복 횟수가 약 $n \log n$ 번으로 줄어들어 1초 제한을 넉넉하게
  통과할 수 있습니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

작성해주신 로직은 다음과 같은 탄탄한 흐름을 가지고 있습니다.

```markdown
1. n을 입력받는다.
2. 중복을 걸러낼 딕셔너리(result)를 준비한다.
3. 1부터 n까지 가로(row)와 세로(col)를 2중 for문으로 순회한다.
4. 가로 * 세로 넓이가 n을 초과하면 더 탐색할 필요가 없으므로 break로 빠져나온다.
5. 넓이가 n 이하일 때, 직사각형을 90도 회전시킨 (세로, 가로) 쌍이 딕셔너리에 없다면,
6. 현재 (가로, 세로) 쌍을 딕셔너리에 등록한다.
7. 최종적으로 딕셔너리에 담긴 고유한 직사각형의 개수를 출력한다.
```

### 3. 💻 코드 구현 리뷰

실제 사물을 다루듯 "뒤집었을 때 이미 있는 모양인지 확인한다"는 현실적인 비유가 로직에 그대로 녹아있어 가독성이 훌륭합니다!

다만, 공간 복잡도(메모리) 측면에서 아주 살짝 아쉬운 점이 있습니다. 딕셔너리는 값을 찾는 속도는 매우 빠르지만, 입력되는 $n$이 100만, 1000만 단위로 커진다면 딕셔너리에 튜플을 계속 저장하면서 메모리
초과가 발생할 위험이 있습니다.

### 4. 🛡️ 엣지 케이스 점검

* **$n=1$ 인 경우:** $1 \times 1$ 정사각형 1개가 정확히 계산되어 딕셔너리에 담기므로 완벽하게 통과합니다.
* **시간 초과 방어:** 2중 for문 내부에서 `row * col > n` 일 때 `break`를 건 것이 핵심 '가지치기'였습니다. 만약 이 조건이 없었다면 불필요하게 1억
  번($10000 \times 10000$)을 돌면서 시간 초과가 났을 텐데, 이 위험을 정확히 회피하셨습니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

이 문제는 메모리를 차지하는 딕셔너리 없이, **수학적 제약 조건** 하나만 추가하면 메모리 사용량을 `O(1)`로 줄이고 속도도 비약적으로 끌어올릴 수 있습니다.

"가로 길이가 세로 길이보다 길어지는 순간은 무조건 '회전된 중복'이다"라는 규칙을 세워, **애초에 가로가 세로보다 작거나 같을 때만 탐색**하도록 만드는 것입니다.

```python
import sys

input = sys.stdin.readline

n = int(input())
count = 0

# 가로 길이(row)는 세로 길이(col)보다 작거나 같다고 강제합니다.
# 따라서 가로 길이의 최대치는 n의 제곱근을 넘을 수 없습니다. (ex. n=10일 때 가로는 3까지만 확인)
for row in range(1, int(n ** 0.5) + 1):
    # 세로 길이는 가로 길이부터 시작해서, 넓이가 n 이하일 때까지만 경우의 수에 더합니다.
    # n // row 는 현재 가로 길이로 만들 수 있는 최대 세로 길이를 의미합니다.
    count += (n // row) - row + 1

print(count)
```

💡 **최적화 핵심 원리:**

* **루프 범위 축소 (제곱근 활용):** 가로(`row`)를 기준으로 세로(`col`)를 탐색할 때, `row`가 $\sqrt{n}$을 넘어가면 무조건 이전 탐색과 중복됩니다. 따라서 바깥쪽 for문의 범위를
  `int(n**0.5) + 1`까지만 제한합니다.
* **수학적 계산으로 내부 루프 제거:** 내부적으로 `col`을 일일이 돌며 세는 대신, `(n // row) - row + 1` 식을 사용하면 현재 `row`에서 만들 수 있는 직사각형의 개수를 단 한 번의
  사칙연산으로 구할 수 있습니다.
