### 1. 🔍 요구사항 분석 및 유형 파악
* **문제 유형:** 완전 탐색 (Brute-Force), 조합 (Combinations)
* **유형 파악 키워드:** "N장의 카드 중에서 3장의 카드를 고른다", "$N$은 100 이하이다." (데이터의 크기가 매우 작아 모든 경우의 수를 전부 확인해도 시간 초과가 나지 않음을 암시하는 핵심 단서입니다.)
* **핵심 목표:** 3장의 카드 합이 $M$을 넘지 않으면서 $M$에 가장 가까운 최댓값을 구하기
* **입력 범위 및 시간 복잡도:** $N$은 최대 100입니다. 100장 중 3장을 고르는 경우의 수는 약 16만 번으로, 파이썬의 초당 연산 횟수(약 2,000만 번) 내에 충분히 들어오므로 $O(N^3)$ 접근이 가능합니다.
* **주의할 엣지 케이스:** 합이 정확히 $M$이 되는 경우 더 이상 탐색할 필요가 없으므로 즉시 종료하여 시간을 단축할 수 있습니다.

### 2. 🧠 논리적 설계 (주석 뼈대)
코드를 작성하기 전, 파이썬에서는 다음과 같은 흐름으로 머릿속에 뼈대를 잡는 것이 가장 효율적입니다.
```markdown
1. N(카드 개수)과 M(목표 합)을 입력받고, 카드 번호 리스트를 초기화한다.
2. 3장의 카드를 고르는 모든 경우의 수(조합)를 순회한다.
3. 고른 3장의 카드 합을 구한다.
4. 그 합이 M 이하이면서 동시에 기존에 저장된 최댓값보다 크면, 최댓값을 갱신한다.
5. 만약 방금 갱신한 합이 M과 완벽히 일치한다면, 더 이상 탐색할 필요가 없으므로 즉시 종료(break)한다.
6. 최종 갱신된 최댓값을 출력한다.
```

### 3. 💻 코드 구현 리뷰
3중 `for`문의 인덱스를 각각 `N-2`, `N-1`, `N`으로 설정하여 동일한 카드를 중복해서 뽑지 않도록 촘촘하게 제어한 로직이 아주 탄탄합니다! 다만, 파이썬 환경의 이점을 살리기 위해 두 가지 정도 아쉬운 점을 짚어드립니다.

* **불필요한 모듈 호출:** `cmath.inf`를 가져오셨지만, 주어지는 카드는 모두 양의 정수이므로 3장의 합 역시 무조건 양수입니다. 따라서 외부 모듈 없이 `result = 0`으로만 초기화해도 충분히 정상 동작합니다.
* **중복 연산 발생:**
    ```python
    if result < cards[i] + cards[j] + cards[k] <= M:
        result = cards[i] + cards[j] + cards[k]
    ```    
    이 부분에서 카드 3장의 합을 `if` 조건문에서 한 번, `result` 변수에 갱신할 때 또 한 번 계산하고 있습니다. 이를 `card_sum = cards[i] + cards[j] + cards[k]`처럼 변수에 미리 담아두고 사용하면 연산 횟수를 절반으로 줄일 수 있습니다.


### 4. 🛡️ 엣지 케이스 점검
* 작성하신 `result < ... <= M` 조건식 덕분에 $M$을 초과하는 합산 결과는 완벽하게 필터링되고 있습니다.
* 문제의 조건상 카드가 최소 3장 주어지기 때문에, 반복문 내에서 인덱스 초과(IndexError)가 발생할 위험 구간도 안전하게 방어되었습니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)
파이썬에서는 3중 `for`문을 직접 타이핑하는 대신, 표준 라이브러리인 **`itertools.combinations`** 를 활용하면 코드를 훨씬 직관적이고 빠르게 만들 수 있습니다. 여기에 시간 단축을 위한 조기 종료(`break`) 조건까지 더한 최적화 코드는 다음과 같습니다.

```python
import sys
from itertools import combinations

# 빠른 입출력 적용
input = sys.stdin.readline
N, M = map(int, input().split())
cards = list(map(int, input().split()))

result = 0

# combinations를 활용해 카드 3장을 뽑는 모든 경우의 수를 탐색
for comb in combinations(cards, 3):
    card_sum = sum(comb)
    
    # 합이 M을 넘지 않으면서 기존 최댓값보다 클 때 갱신
    if result < card_sum <= M:
        result = card_sum
        
        # 최적화 포인트: 목표치 M과 완전히 같아지면 즉시 탐색 종료
        if result == M:
            break

print(result)
```
**💡 최적화 핵심:**
직접 인덱스를 컨트롤하는 수고를 덜어주고 가독성을 비약적으로 높여주는 `combinations` 라이브러리의 활용, 그리고 불필요한 반복을 끊어내는 `break`가 파이썬 코딩테스트 시간 단축의 핵심 무기입니다.
