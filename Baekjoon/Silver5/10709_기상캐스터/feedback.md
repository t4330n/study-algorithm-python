### 1. 🔍 요구사항 분석 및 유형 파악 (Upgrade!)

* **문제 유형:** 2차원 배열 시뮬레이션, 상태 변수(Flag) 탐색
* **유형 파악 키워드:** "1분마다 1킬로미터씩 동쪽(오른쪽)으로 이동한다", "처음 구름이 뜬 위치" (오른쪽으로만 이동하므로 왼쪽 칸의 영향을 받는다는 것이 핵심 단서입니다.)
* **핵심 목표:** 각 칸을 기준으로, 자기보다 왼쪽에 있는 가장 가까운 구름('c')과의 거리를 구하기.
* **입력 범위 및 시간 복잡도:** $H, W$는 최대 100입니다. 전체 칸의 개수가 10,000개이므로, 이중 for문 한 번 $O(H \times W)$으로 1초 제한(약 2천만 번 연산) 안에 아주
  여유롭게 통과합니다.
* **주의할 엣지 케이스:**
    * 해당 행에 구름이 아예 없는 경우 (모두 -1이 나와야 함)
    * 한 행에 구름이 여러 개 있는 경우 (새로운 구름을 만나면 거리가 다시 0으로 초기화되어야 함)

### 2. 🧠 논리적 설계 (주석 뼈대)

각 행(가로줄)은 구름 이동에 있어 서로 전혀 영향을 주지 않는 독립적인 세계입니다. 따라서 2차원 배열에 전체를 담을 필요 없이, 한 줄씩 입력받아 곧바로 처리하는 뼈대를 세웁니다.

```markdown
1. H(세로), W(가로)를 입력받는다.
2. H번 반복하며 각 행의 문자열을 입력받는다. (행 단위로 독립 계산)
3. '구름이 지난 시간(cloud_timer)' 상태 변수를 -1로 초기화한다.
4. 문자열을 한 글자씩 왼쪽부터 순회한다:
   4-1. 구름('c')을 만나면 cloud_timer를 0으로 세팅한다.
   4-2. 빈 하늘('.')을 만났는데, cloud_timer가 -1이 아니라면 1을 더한다.
5. 각 칸마다 기록된 cloud_timer 값을 리스트에 담아 즉시 언패킹(*) 출력한다.
```

### 3. 💻 코드 구현 (Pythonic)

작성하신 코드는 2차원 배열을 만들고, 0과 -1을 먼저 세팅한 뒤, `result[row][col - 1] >= 0`을 통해 왼쪽 칸을 참조하여 거리를 더해가는 직관적이고 훌륭한 로직입니다! 특히 마지막에
`print(*row)`로 출력하신 부분은 파이썬의 언패킹을 아주 적절하게 사용하셨습니다.

**[아쉬운 점 & 최적화 포인트]**
하지만 파이썬의 이점을 살리면 굳이 3번이나 전체 배열을 순회할(루프를 나눌) 필요가 없습니다.
입력을 2차원 리스트(`cloud`)로 전부 저장하고, `result` 빈 배열을 만들고, 그걸 채우고, 또 순회하는 대신 **입력을 한 줄 받을 때마다 즉시 계산해서 출력하고 버리면** 메모리와 시간을
절반 이하로 확 줄일 수 있습니다.

### 4. 🛡️ 엣지 케이스 점검

* **해당 행에 구름이 아예 없는 경우:** 사용자님의 코드에서 `result` 초기값을 `-1`로 세팅해 두었기 때문에 두 번째 루프의 `result[row][col-1] >= 0` 조건을 절대 통과하지 못해
  끝까지 `-1`로 완벽하게 유지됩니다.
* **새로운 구름 등장 시 덮어쓰기 방어:** 첫 번째 루프에서 구름 위치를 `0`으로 강력하게 고정해 두었고, 두 번째 루프에서는 `if result[row][col] == -1` 일 때만 값을 더하도록
  막아두셨기 때문에 새로운 구름이 등장해도 거리가 0부터 다시 잘 시작됩니다. 방어가 아주 튼튼합니다!

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

행 간의 간섭이 없다는 점을 이용하여 **'상태 변수(cloud_timer)를 활용한 1차원 실시간 처리'** 로 루프와 메모리 사용량을 극한으로 줄인 최적화 코드입니다.

```python
import sys

input = sys.stdin.readline

H, W = map(int, input().split())

# 2차원 배열을 만들지 않고, 한 줄을 입력받자마자 바로 처리합니다.
for _ in range(H):
    row_data = input().strip()
    result = []

    # 구름과의 거리를 추적하는 상태 변수 (구름을 만나기 전까진 -1 유지)
    cloud_timer = -1

    for char in row_data:
        if char == 'c':
            cloud_timer = 0  # 구름을 만나면 즉시 0으로 초기화
        elif cloud_timer != -1:
            cloud_timer += 1  # 이전에 구름을 만난 적이 있다면 시간 증가

        result.append(cloud_timer)

    # 한 행의 처리가 끝날 때마다 즉시 출력
    print(*result)
```

💡 **최적화 핵심 원리:**

이 기법은 배열의 요소를 직접 참조(`result[row][col-1]`)하는 대신, 별도의 상태 변수(`cloud_timer`) 하나가 왼쪽의 값을 '기억'하면서 오른쪽으로 이동하는 구조입니다.
인덱스를 직접 제어하며 발생할 수 있는 오류를 없애주고, 2차원 전체를 훑는 불필요한 연산을 없애주어 시뮬레이션 코드의 길이를 가장 압도적으로 줄여주는 파이써닉한 테크닉입니다.
