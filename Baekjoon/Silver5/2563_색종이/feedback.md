링크만 남겨주신 걸 보니, 이번 문제는 수학적인 교집합/합집합 생각에 빠져 어떻게 접근해야 할지 감을 잡기 위해 넘겨주셨군요!

이 문제는 겹치는 넓이를 수식으로 빼려고 하면 경우의 수가 너무 많아져서 머리가 아파지지만, **'컴퓨터의 시선(2차원 배열)'** 으로 바라보면 정말 허무할 정도로 쉽게 풀리는 아주 유명한 '시뮬레이션'의 기초
문제입니다.

약속된 5단계로 패턴을 완벽히 부숴드리겠습니다.

---

### 1. 🔍 요구사항 분석 및 유형 파악

* **문제 유형:** 2차원 배열 (2D Array), 시뮬레이션 (Simulation)
* **유형 파악 키워드:** "도화지", "색종이를 붙인 후", "검은 영역의 넓이"
* **핵심 목표:** 여러 장의 색종이가 겹치든 말든, 최종적으로 색종이가 덮인 구역의 총넓이를 구하기.
* **입력 범위 및 시간 복잡도:** 색종이의 수 $N$은 100 이하이고, 도화지의 크기는 100x100으로 고정되어 있습니다. 도화지 전체 칸 수는 10,000개밖에 되지 않으므로, 색종이를 붙일 때마다 배열을
  탐색하는 $O(N \times 100)$ 알고리즘을 써도 시간 제한(1초) 내에 아주 여유롭게 통과합니다.
* **주의할 엣지 케이스:** 색종이가 2장 겹칠 때, 3장 겹칠 때, 4장 겹칠 때를 수식으로 예외 처리하려고 하면 코드가 꼬입니다. 겹치는 횟수와 상관없이 '색칠된 상태'만 유지하는 것이 핵심입니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

수학적으로 접근하지 않고, 진짜 현실에서 **100x100 모눈종이에 검은색 펜으로 색칠을 한다**고 상상해 보세요. 이미 검게 칠해진 곳 위에 또 검은색을 칠해도 어차피 검은색입니다. 이것이 핵심 아이디어입니다.

```markdown
1. 가로 100, 세로 100 크기의 2차원 도화지 배열을 0으로 초기화한다.
2. 색종이의 개수 N을 입력받는다.
3. N번 반복하며 색종이의 좌측 하단 (x, y) 좌표를 입력받는다.
4. 입력받은 x부터 x+10까지, y부터 y+10까지 2중 for문을 돈다.
5. 해당 도화지 좌표의 값을 1로 바꾼다. (이미 1이어도 그냥 1을 덮어씌운다 -> 겹침 문제 해결!)
6. 모든 색종이를 붙인 후, 도화지 배열 전체를 순회하며 1의 개수를 모두 더해 출력한다.
```

### 3. 💻 코드 구현 (Pythonic)

뼈대를 바탕으로 파이썬의 리스트 컴프리헨션을 활용하여 아주 깔끔하게 구현한 코드입니다.

```python
import sys

input = sys.stdin.readline

# 1. 100x100 도화지 0으로 초기화
board = [[0] * 100 for _ in range(100)]

N = int(input())

# 2. N개의 색종이 붙이기 시뮬레이션
for _ in range(N):
    x, y = map(int, input().split())

    # 3. 해당 색종이의 영역(10x10)을 모두 1로 칠하기
    for i in range(x, x + 10):
        for j in range(y, y + 10):
            board[i][j] = 1  # 겹치더라도 1로 덮어씌워짐

# 4. 2차원 배열에 있는 1의 총 개수(넓이) 구하기
total_area = 0
for row in board:
    total_area += sum(row)

print(total_area)
```

### 4. 🛡️ 엣지 케이스 점검

* **색종이가 도화지 밖으로 나가는 경우:** 문제 조건에서 "색종이가 도화지 밖으로 나가는 경우는 없다"고 명시되어 있으므로, 배열의 인덱스 초과(IndexError)는 발생하지 않습니다.
* **여러 장이 완전히 겹치는 경우:** `board[i][j] += 1`이 아니라 `board[i][j] = 1`을 사용했기 때문에, 100장이 같은 자리에 겹쳐도 넓이는 정확히 100으로 유지됩니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

위의 코드도 충분히 훌륭하고 직관적입니다. 하지만 파이썬의 특성을 극대화하면, 마지막에 넓이를 구하는 `for`문을 한 줄로 압축하여 코드를 더 짧고 우아하게 만들 수 있습니다.

**[리스트 언패킹과 1차원 합축을 이용한 최적화]**

```python
import sys

input = sys.stdin.readline
board = [[0] * 100 for _ in range(100)]

for _ in range(int(input())):
    x, y = map(int, input().split())
    for i in range(x, x + 10):
        for j in range(y, y + 10):
            board[i][j] = 1

# 2차원 리스트 내부의 리스트들을 하나씩 꺼내어 sum()을 적용하고, 그 결과들을 다시 sum()
print(sum(sum(row) for row in board))
```

**💡 최적화 핵심 원리:**
이런 방식의 풀이를 알고리즘에서는 **'상태 배열(Boolean Array)을 활용한 시뮬레이션'** 이라고 부릅니다.
복잡한 수리적 계산이나 기하학적 교집합 공식(`A + B - (A ∩ B)`)을 코드로 구현하려다 보면 실수하기 십상입니다. 하지만 메모리가 허락하는 한, 이렇게 가상의 도화지를 만들어 직접 색칠해 버리는 방식이
코딩테스트에서는 가장 강력하고 예외 없는 무적의 치트키가 됩니다. 앞으로 2차원 평면에서 영역이 겹치는 문제가 나오면 무조건 **"격자 배열을 만들고 1로 칠한다!"** 를 떠올리시면 됩니다.
