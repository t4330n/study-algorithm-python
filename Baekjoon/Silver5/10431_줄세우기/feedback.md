제공된 예제 테스트 케이스는 다 맞는데 제출하면 "틀렸습니다"가 뜰 때의 그 허탈함, 정말 뼈저리게 공감합니다! 코딩테스트를 준비하며 누구나 한 번쯤 겪는 아주 전형적인 **'배열 조작 중 인덱스 꼬임(Index
Shifting)'** 현상에 당하셨습니다.

어디서 꼬였는지, 그리고 이 복잡한 문제를 어떻게 발상의 전환으로 한 번에 부술 수 있는지 약속된 5단계로 확실하게 분석해 드립니다.

---

### 1. 🔍 요구사항 분석 및 유형 파악

* **문제 유형:** 정렬 (Insertion Sort), 시뮬레이션, **역전 수(Inversion Count)**
* **유형 파악 키워드:** "학생들이 도착하는 순서대로", "자기보다 큰 학생이 있으면 그 앞에 선다", "물러난 걸음 수"
* **핵심 단서:** 데이터의 크기가 정확히 20명으로 고정되어 있습니다. 시간 초과는 절대 발생하지 않으므로 $O(N^2)$ 이상의 완전 탐색을 마음껏 써도 됩니다.
* **발상의 전환 (가장 중요!):** 배열의 요소를 직접 뺐다가(remove) 끼워 넣을(insert) 필요가 **전혀 없습니다.** 새로운 학생이 줄에 설 때, 뒤로 물러나는 걸음 수는 결국 **"이미 줄을 서
  있는 사람 중 나보다 키가 큰 사람의 수"** 와 완벽하게 동일합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

학생들이 '순서대로' 들어온다는 문제의 흐름에 맞춰, 왼쪽에서 오른쪽으로 진행하는 정방향 뼈대를 세웁니다.

```markdown
1. 테스트 케이스의 수 P를 입력받는다.
2. 각 케이스마다 [테스트 번호]와 [20명의 학생 키]를 분리한다.
3. 첫 번째 학생(인덱스 0)부터 마지막 학생(인덱스 19)까지 순서대로 세운다.
4. '현재 학생(i)'을 기준으로, '내 앞에 서 있는 학생들(0 ~ i-1)'과 키를 비교한다.
5. 내 앞에 나보다 키가 큰 사람이 있다면, 그 사람의 수만큼 내가 걸어가며 사람들을 뒤로 밀어내야 한다.
6. 따라서 (나보다 키 큰 사람의 수)를 result에 누적해서 더해준다.
```

### 3. 💻 코드 구현 리뷰 (틀린 이유)

테스트 케이스는 통과했지만 채점기에서 오답이 나온 치명적인 이유는, **배열을 뒤에서부터(역방향) 탐색하며 요소를 넣고 빼는 과정에서 인덱스가 어긋났기 때문**입니다.

* **사용자님 코드의 논리적 오류 추적:**
  만약 키가 `[3, 2, 1]` 순서로 들어왔다고 가정해 봅시다. (정답은 3번 물러나야 함)
    1. `pointer = 2` (키 1) -> 앞의 `3, 2`가 더 크므로 맨 앞으로 이동. 배열은 `[1, 3, 2]`가 됨.
    2. **여기서 문제가 발생합니다!** `pointer -= 1`이 되면서 `pointer = 1`을 가리킵니다.
    3. 현재 배열 `[1, 3, 2]`에서 1번 인덱스는 키 `3`입니다.
    4. 원래라면 키 `2`를 검사했어야 했는데, 1번이 맨 앞으로 끼어들어 오면서 기존의 사람들이 오른쪽으로 밀려났고, 그 결과 `pointer`가 검사해야 할 대상을 건너뛰게(Skip) 된 것입니다.

예제 테스트 케이스는 운 좋게 이러한 인덱스 건너뜀이 발생하지 않는 패턴이었을 뿐, 숨겨진 반례에서는 여지없이 무너지는 구조였습니다.

### 4. 🛡️ 엣지 케이스 점검

* **이미 오름차순으로 정렬된 경우:** 내 앞에 나보다 큰 사람이 한 명도 없으므로 result는 0이 됩니다.
* **완벽한 내림차순인 경우:** 매번 들어올 때마다 맨 앞까지 걸어가야 하므로, 총 걸음 수는 $1 + 2 + 3 + ... + 19 = 190$이 되어 정상적으로 최댓값을 구합니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

배열을 직접 자르고 붙이는(`remove`, `insert`) 행위를 아예 없애버리고, 수학적 발상(내 앞의 큰 사람 수 세기)을 파이썬 2중 for문으로 깔끔하게 구현한 최적화 코드입니다.

```python
import sys

input = sys.stdin.readline

P = int(input())

for _ in range(P):
    data = list(map(int, input().split()))
    t = data[0]
    students = data[1:]

    result = 0
    # 20명의 학생을 순서대로 입장시킵니다.
    for i in range(20):
        # j는 '현재 학생(i)보다 먼저 줄을 선 학생들'을 의미합니다.
        for j in range(i):
            # 내 앞에 서 있는 사람(j)이 나(i)보다 키가 크다면?
            if students[j] > students[i]:
                # 그 사람이 한 칸 뒤로 밀려나야 하므로 걸음 수 1 증가!
                result += 1

    print(f'{t} {result}')
```

💡 **최적화 핵심 원리:**

이 문제는 시뮬레이션의 탈을 쓴 **"버블 정렬(Bubble Sort) 또는 삽입 정렬(Insertion Sort)의 Swap 횟수 구하기"** 문제입니다.
배열의 원소를 직접 움직이는 시뮬레이션은 코드가 복잡해지고 인덱스 에러를 유발하기 쉽습니다. "A가 B보다 앞으로 가기 위해 뒤로 물러나야 하는 총 횟수 = A앞에 있는 큰 사람의 수"라는 상태값만 추출해서
더해버리면, 버그가 발생할 확률 0%의 무적 코드가 완성됩니다!
