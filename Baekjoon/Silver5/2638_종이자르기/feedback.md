### 1. 🔍 요구사항 분석 및 유형 파악

* **문제 유형:** 정렬 (Sorting), 그리디 (Greedy)
* **유형 파악 키워드:** "가장 큰 종이 조각의 넓이", "잘리는 점선"
    * 2차원 시뮬레이션으로 풀면 너무 복잡해집니다. 가로와 세로를 완전히 **독립적인 1차원 선분**으로 나누어 생각하는 것이 이 유형의 핵심 파훼법입니다.
* **핵심 목표:** 가로 절취선과 세로 절취선을 나누어 정렬한 뒤, 인접한 선들 사이의 '간격' 중 최댓값 구하기.
* **입력 범위 및 시간 복잡도:** 가로, 세로의 최대 길이는 100, 자르는 횟수는 최대 100입니다. $O(N \log N)$의 정렬 속도와 한 번의 순회만으로 해결되므로 시간 초과 위험은 0%입니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

작성해주신 완벽한 로직을 실전 뼈대로 정리하면 다음과 같습니다.

```markdown
1. 종이의 가로, 세로 길이를 입력받는다.
2. 가로 자르기 위치와 세로 자르기 위치를 각각 다른 리스트에 저장한다.
   (이때, 양 끝점인 0과 종이의 끝 길이를 리스트에 미리 포함해 둔다!)
3. 절취선 리스트들을 오름차순으로 정렬한다.
4. 정렬된 리스트를 순회하며, 인접한 두 점 사이의 길이(간격)를 구한다.
5. 가로 간격 중 최댓값과 세로 간격 중 최댓값을 곱하여 출력한다.
```

### 3. 💻 코드 구현 리뷰

종이의 양 끝점을 배열에 추가해 둔 덕분에 예외 처리 없이 아주 깔끔한 흐름이 만들어졌습니다. 다만, 자른 후의 길이를 구하는 반복문 로직이 살짝 복잡하게 꼬여 있습니다.

```python
# 사용자님 코드
for i in range(len(sgl) - 1, 0, -1):
    for j in range(i - 1, i - 2, -1):
        igl.append(sgl[i] - sgl[j])
```

이 2중 for문에서 내부의 `j`는 결국 `i - 1` 단 하나만 가리키게 됩니다. 즉, 굳이 2중 for문이나 역순(`-1`) 탐색을 쓸 필요 없이, 정방향으로 `sgl[i] - sgl[i-1]`을 해주면 훨씬
직관적이고 알아보기 쉬운 코드가 됩니다.

### 4. 🛡️ 엣지 케이스 점검

* **종이를 한 번도 자르지 않는 경우 ($N=0$):**
  입력으로 0이 들어오더라도 사용자님의 코드에서는 초기에 세팅한 `[0]`과 `[끝 길이]` 배열이 존재합니다. 따라서 반복문을 돌면 `끝 길이 - 0 = 전체 길이`가 자연스럽게 도출되어, 자르지 않은 원본
  종이의 넓이가 무사히 출력됩니다. 아주 견고한 예외 처리입니다!

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

리뷰 내용을 바탕으로, 불필요한 2중 for문을 걷어내고 파이썬의 **제너레이터(Generator) 표현식**을 활용해 추가 리스트(`igl`, `isl`) 생성 없이 메모리와 코드를 압축한 최적화 버전입니다.

이러한 리스트 컴프리헨션 및 내포 문법은 노션에 잘 정리해 두시면 실전 코딩테스트에서 타자 시간을 확 줄여주는 강력한 무기가 됩니다.

```python
import sys

input = sys.stdin.readline

width, height = map(int, input().split())
N = int(input())

# 1. 양 끝점(0, 전체 길이)을 미리 넣어서 리스트 생성
horizontal_cuts = [0, height]
vertical_cuts = [0, width]

# 2. 절취선 위치 입력받기
for _ in range(N):
    direction, location = map(int, input().split())
    if direction == 0:
        horizontal_cuts.append(location)
    else:
        vertical_cuts.append(location)

# 3. 오름차순 정렬
horizontal_cuts.sort()
vertical_cuts.sort()

# 4. [최적화 핵심] 인접한 두 절취선 사이의 간격 중 최댓값을 한 번에 구하기
# 추가 리스트(append) 없이 루프를 돌면서 곧바로 최댓값을 뽑아냅니다.
max_h = max(horizontal_cuts[i] - horizontal_cuts[i - 1] for i in range(1, len(horizontal_cuts)))
max_v = max(vertical_cuts[i] - vertical_cuts[i - 1] for i in range(1, len(vertical_cuts)))

print(max_h * max_v)
```

💡 **최적화 핵심 원리:**

1. **제너레이터 표현식 (Generator Expression):** `max(...)` 괄호 안에 들어간 코드를 자세히 보시면 리스트 컴프리헨션과 달리 대괄호 `[ ]`가 없습니다. 이는 잘린 길이들을 담아둘
   새로운 리스트를 메모리에 굳이 만들지 않고, 값을 하나씩 생성하면서 곧바로 최댓값을 갱신해 나가는 파이썬의 고급 기법입니다. 공간 복잡도(메모리)를 극단적으로 줄이는 데 효과적입니다.
2. **독립적 차원 분리 (그리디 접근):** 2차원 평면의 넓이를 구하는 문제지만, "가장 큰 사각형의 넓이 = 가로 길이의 최댓값 $\times$ 세로 길이의 최댓값"이라는 수학적 사실을 간파한 것이
   핵심입니다. 복잡한 2차원 시뮬레이션을 1차원 선분 탐색 2개로 완벽하게 분리하여 $O(N \log N)$의 정렬 속도만으로 문제를 끝내버렸습니다.
3. **정방향 1차원 순회:** `sgl[i] - sgl[i-1]` 처럼 정렬된 배열에서 바로 앞 인덱스와의 차이를 구하는 방식은 인덱스 오류를 방지하고 가독성을 크게 높여주는 가장 파이써닉한 탐색 패턴 중
   하나입니다.
