### 1. 🔍 요구사항 분석 및 유형 파악 (Upgrade!)
* **문제 유형:** 1차원 배열 시뮬레이션, 수학 (모듈러 연산)
* **유형 파악 키워드:** "1초에 1미터 이동", "빨간색 불일 때 대기, 초록색 불일 때 이동", "주기적으로 반복" 
    * 현재 누적된 **'시간'** 을 기준으로 상태를 판단해야 하는 전형적인 흐름 제어 문제입니다.
* **핵심 목표:** 도착점 $L$까지 이동하면서, 신호등을 만났을 때의 '누적 시간'을 계산해 대기 시간을 더해주기.
* **입력 범위 및 시간 복잡도:** $L \le 1000$, $N \le 100$. 거리가 매우 짧으므로 1미터씩 이동하며 $O(L)$ 탐색을 해도 충분하고, 신호등 위치로만 점프하는 $O(N)$ 최적화도 가능한 아주 넉넉한 1초 제한입니다.
* **주의할 엣지 케이스:** 아무 신호등에도 걸리지 않고(모두 초록불) 무사통과하는 경우. 이때 정답은 반드시 거리 $L$과 똑같이 나와야 합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)
1미터씩 검사하는 배열 방식에서 벗어나, **"신호등이 있는 곳으로 순간이동"** 하는 방식으로 발상을 전환하면 로직이 압도적으로 직관적이게 변합니다.

```markdown
1. 신호등 개수 N과 도로 길이 L을 입력받는다.
2. 현재 위치(current_pos)와 누적 시간(current_time)을 0으로 초기화한다.
3. N개의 신호등 정보를 순서대로 순회하며:
   3-1. (다음 신호등 위치 - 현재 위치) 만큼 이동 시간을 더해준다.
   3-2. 현재 위치를 해당 신호등 위치로 갱신한다.
   3-3. 현재 누적 시간 기준으로 신호등의 주기(R + G)를 나눈 나머지를 구한다.
   3-4. 나머지가 빨간불 시간(R)보다 작다면, 남은 빨간불 시간만큼 누적 시간에 더해준다(대기).
4. 마지막 신호등을 지난 후, 도착점(L)까지 남은 거리만큼 시간을 마저 더하고 출력한다.
```

### 3. 💻 코드 구현 리뷰 (치명적인 논리 버그 발견!)
사용자님의 코드가 예제 1번(정답 12)을 통과한 것은 사실 **엄청난 수학적 우연**이 겹쳤기 때문입니다! 

* **버그의 원인: `for idx in range(len(road)): result += 1`**
    * 사용자님이 만든 `road` 배열의 길이는 `L + 1`입니다. ($L=10$이면 길이가 11)
    * 따라서 저 반복문은 11번 돕니다. 만약 신호등이 하나도 없었다면 거리가 10이므로 시간도 10이 나와야 하는데, 이 코드는 무조건 **기본값 11**을 출력하게 됩니다.
* **그런데 예제는 왜 맞았을까?**
    * 반복문 때문에 신호등 검사를 실제 도착 시간보다 '1초 늦게' 판단하게 되었습니다. 
    * 그런데 우연히 예제의 신호등 주기에서 "1초 늦게 검사했더니 -> 대기해야 할 시간이 정확히 1초 줄어드는" 구간에 걸려버렸습니다.
    * 즉, **늘어난 이동 시간(+1)과 줄어든 대기 시간(-1)이 상쇄**되어 기적적으로 정답 12가 나온 것입니다. 숨겨진 반례(초록불 통과 등)가 들어오면 여지없이 오답이 나오는 구조입니다.

### 4. 🛡️ 엣지 케이스 점검
* **배열 초기화 함정 방어:** 사용자님의 코드는 빈 공간을 0으로 채웠습니다. 만약 `sum(road[idx])`를 할 때 요소가 int형 0이었다면 에러가 났겠지만, 센스 있게 `if road[idx] != 0:` 이라는 조건으로 완벽하게 방어해 내셨습니다.
* **초록불 도착 시 엣지 케이스:** 위에서 언급했듯 아무 대기 없이 통과할 때 거리보다 +1이 더 나오는(Off-by-one) 치명적인 엣지 케이스를 놓쳤습니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)
배열 자체를 만들 필요 없이, 설계한 뼈대대로 **신호등 사이를 $O(N)$ 속도로 껑충껑충 뛰어넘는** 가장 이상적인 파이써닉 정답 코드입니다.

```python
import sys
input = sys.stdin.readline

N, L = map(int, input().split())

current_time = 0
current_pos = 0

for _ in range(N):
    D, R, G = map(int, input().split())
    
    # 1. 이전 위치에서 현재 신호등까지 이동한 시간 더하기
    current_time += (D - current_pos)
    current_pos = D  # 트럭 위치 갱신
    
    # 2. 신호등 주기(cycle) 확인
    cycle = R + G
    state = current_time % cycle
    
    # 3. 현재 주기가 빨간불(R) 구간에 있다면, 파란불이 켜질 때까지 대기
    if state < R:
        current_time += (R - state)

# 4. 마지막 신호등에서 도착점(L)까지 남은 거리만큼 시간 추가
current_time += (L - current_pos)

print(current_time)
```

💡 **최적화 핵심 원리:**

* **공간 최적화 (배열 제거):** `road` 배열을 만들지 않고 주어지는 입력 데이터만 즉시 처리 후 버리기 때문에 메모리 사용량이 극도로 최소화됩니다.
* **시간 도약 (1미터 시뮬레이션 -> 체크포인트 점프):** 1미터씩 일일이 검사하는 $O(L)$ 로직에서, 수학적 거리 연산(`D - current_pos`)을 통해 한 번에 순간 이동하는 $O(N)$ 로직으로 진화했습니다. 데이터가 커질수록 이 방식이 압도적인 속도 차이를 만들어냅니다.
