### 1. 🔍 요구사항 분석 및 유형 파악 (Upgrade!)

* **문제 유형:** 그리디 알고리즘 (Greedy), 수학
* **유형 파악 키워드:** "최소 개수의 봉지", "정확하게 N킬로그램", "3킬로그램 봉지와 5킬로그램 봉지" 
    * 여러 옵션 중 가장 큰 단위(5kg)를 우선적으로, 그리고 최대한 많이 사용해야 최적해를 보장한다는 전형적인 그리디의 단서입니다.
* **핵심 목표:** $5x + 3y = N$을 만족하는 $x+y$의 최솟값을 찾기. 불가능할 경우 -1 출력.
* **입력 범위 및 시간 복잡도:** $N$은 최대 5,000입니다. 시간 제한이 1초이므로 1씩 줄여나가는 $O(N)$ 탐색이나 수학적 $O(1)$ 연산 모두 아주 여유롭게 통과합니다.
* **주의할 엣지 케이스:** $N=4$나 $N=7$처럼 3과 5의 조합으로 절대 만들 수 없는 '정확히 나누어 떨어지지 않는 수'에 대한 예외 처리.

### 2. 🧠 논리적 설계 (주석 뼈대)

큰 봉지를 최대한 많이 써야 하므로, **"5로 나누어 떨어질 때까지 3을 빼보는"** 역발상 백트래킹 방식이 가장 직관적이고 확실합니다.

```markdown
1. 배달해야 할 설탕 무게 N을 입력받는다.
2. 사용한 봉지 수를 추적할 변수(bags)를 0으로 초기화한다.
3. N이 0 이상일 동안 다음 로직을 무한 반복한다:
   3-1. 만약 현재 N이 5로 나누어 떨어진다면:
        가장 최적의 상태이므로, (N // 5)를 bags에 더하고 정답을 출력한 뒤 루프를 완전히 탈출(break)한다.
   3-2. 5로 나누어 떨어지지 않는다면:
        3kg 봉지를 하나 사용한 것으로 치고 N에서 3을 뺀 뒤, bags를 1 증가시킨다.
4. 만약 N이 음수가 되어 루프가 끝났다면(break를 만나지 못했다면), 
   정확하게 무게를 맞출 수 없다는 뜻이므로 -1을 출력한다.
```

### 3. 💻 코드 구현 (Pythonic)

설계한 뼈대를 파이썬 특유의 `while - else` 문법으로 구현한 가장 깔끔한 정석 코드입니다.

```python
import sys
input = sys.stdin.readline

N = int(input())
bags = 0

while N >= 0:
    # 5로 나누어 떨어지면 최적해 달성
    if N % 5 == 0:
        bags += (N // 5)
        print(bags)
        break
    
    # 안 나누어지면 3kg 봉지 1개 추가 후 남은 무게 확인
    N -= 3
    bags += 1
else:
    # break로 탈출하지 못하고 while 조건(N >= 0)이 거짓이 되어 끝난 경우
    print(-1)
```

### 4. 🛡️ 엣지 케이스 점검

* **만들 수 없는 무게 ($N=4, 7$):**
    * $N=4$: 5의 배수가 아니므로 3을 뺌 ($N=1, bags=1$). $N=1$도 5의 배수가 아니므로 3을 뺌 ($N=-2, bags=2$). $N \ge 0$ 조건을 만족하지 못해 while문이 종료되고 `else` 블록으로 넘어가 `-1`이 정상 출력됩니다.
* **3으로만 이루어진 무게 ($N=6, 9$):**
    * $N=6$: 3을 두 번 빼면 $N=0, bags=2$가 됩니다. 0은 5로 나누어 떨어지므로(`0 % 5 == 0`), 추가 봉지 없이 그대로 $bags=2$를 출력하고 안전하게 종료됩니다. 완벽하게 방어되었습니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

시뮬레이션(반복문) 과정을 거치지 않고, $N$을 5로 나눈 나머지의 규칙성을 파악하여 수학적 공식으로 단 한 번의 연산에 끝내버리는 $O(1)$ 초극단적 숏코딩 최적화입니다.

```python
N = int(input())

# 4와 7은 절대 불가능한 숫자
if N == 4 or N == 7:
    print(-1)
else:
    # 5로 나눈 몫(5kg 봉지)과, 나머지에 따른 3kg 봉지 및 5kg 봉지 조율 계산식
    print(N // 5 + (N % 5) % 2 * 2 + (N % 5) // 2)
```

💡 **최적화 핵심 원리:**

* **패턴의 상수화:** $N$을 5로 나눈 나머지는 `0, 1, 2, 3, 4` 중 하나입니다.
    * 나머지가 1이면 (`N=6, 11...`): 5kg 봉지 하나를 풀고 3kg 두 개(총 6kg)로 바꾸면 됨 $\rightarrow$ 봉지 수 +1
    * 나머지가 2이면 (`N=12, 17...`): 5kg 봉지 두 개를 풀고 3kg 네 개(총 12kg)로 바꾸면 됨 $\rightarrow$ 봉지 수 +2
    * 나머지가 3이면 (`N=8, 13...`): 남은 3kg을 그대로 3kg 봉지 1개로 담으면 됨 $\rightarrow$ 봉지 수 +1
    * 나머지가 4이면 (`N=9, 14...`): 5kg 봉지 하나를 풀고 남은 4kg과 합쳐 9kg을 3kg 세 개로 바꾸면 됨 $\rightarrow$ 봉지 수 +2
* 이러한 $+1, +2$의 증감 패턴을 수학 공식인 `(N % 5) % 2 * 2 + (N % 5) // 2` 로 압축한 것입니다. 실전에서는 직관적인 `while` 문 풀이(3번 코드)가 가장 안전하지만, 속도와 메모리를 극한으로 줄여야 할 때는 이러한 수학적 접근도 가능하다는 점을 알아두시면 좋습니다.
