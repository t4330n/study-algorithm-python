### 1. 🔍 요구사항 분석 및 유형 파악 (Upgrade!)
* **문제 유형:** 1차원 배열 시뮬레이션, 그리디(투 포인터)
* **유형 파악 키워드:** "오목하게 들어간 부분이 없어야 한다", "가장 높은 기둥"
    * 오목한 부분이 없다는 것은, **가장 높은 기둥(봉우리)을 기준으로 왼쪽은 계속 올라가야 하고, 오른쪽도 계속 올라와야(반대 방향 기준) 한다**는 결정적 단서입니다.
* **핵심 목표:** 가장 높은 기둥을 찾아 기준점(`max_L`)으로 삼고, 왼쪽에서 올라가는 면적 + 오른쪽에서 올라오는 면적 + 지붕 꼭대기 면적 구하기.
* **입력 범위 및 시간 복잡도:** 기둥의 개수 $N \le 1,000$, 위치 $L \le 1,000$. 위치의 최댓값이 1000으로 매우 작으므로, 길이가 1001인 배열을 만들어 $O(MAX\_L)$ 탐색을 해도 1초 안에 아주 여유롭게 통과합니다.
* **주의할 엣지 케이스:** 위치 $L$이 1000으로 들어오는 경우. 파이썬 배열은 0부터 시작하므로 인덱스 1000을 담으려면 배열 크기는 반드시 **1001**이어야 합니다 (사용자님이 처음 겪으셨던 인덱스 에러의 원인입니다).

### 2. 🧠 논리적 설계 (주석 뼈대)
사용자님이 스스로 떠올리신 완벽한 로직을 실전 뼈대로 정리했습니다.

```markdown
1. 위치 L이 최대 1000이므로, 높이를 저장할 1001 크기의 1차원 배열을 0으로 초기화한다.
2. 기둥을 입력받으며 배열의 해당 위치에 높이를 저장하고, 동시에 '가장 높은 기둥의 위치(max_L)'와 '높이(max_H)'를 기록한다.
3. [왼쪽 파트] 0부터 max_L 이전까지 순회하며:
   가장 높았던 갱신값(temp)을 유지하며 면적(result)에 더한다.
4. [오른쪽 파트] 1000부터 max_L 이전까지 역순으로 순회하며:
   가장 높았던 갱신값(temp)을 유지하며 면적(result)에 더한다.
5. [지붕] 양쪽에서 더하지 않은 가장 높은 기둥 본체의 면적(max_H)을 마지막에 더해준다.
```

### 3. 💻 코드 구현 리뷰 (중복 로직 축약)
사용자님의 코드가 길고 더럽게 느껴졌던 이유는 빈 공간(`h == 0`)일 때와 기둥이 있을 때(`h > temp` 등)를 일일이 `if-else`로 나누어 분기 처리했기 때문입니다.

```python
# 사용자님 코드
if h == 0:
   result += temp 
else:
    if h > temp:
        result += h
        temp = h
    else:
        result += temp
```

하지만 가만히 생각해 보면, **"기존 최고 높이(`temp`)와 현재 위치의 높이(`h`) 중 더 큰 값을 현재 지붕의 높이로 삼는다"** 는 단 하나의 문장으로 압축됩니다. 파이썬의 `max()` 함수를 쓰면 이 9줄의 코드가 단 2줄로 줄어듭니다.

### 4. 🛡️ 엣지 케이스 점검
* **가장 높은 기둥이 여러 개인 경우:** 첫 번째로 등장하는 최고점 위치를 `max_L`로 삼게 됩니다. 오른쪽에서 역방향으로 올 때, 지붕이 평평하게 유지되어야 하므로 `temp = max(temp, h)` 로직이 알아서 평평한 지붕을 형성하며 최고점까지 도달하므로 완벽하게 방어됩니다.
* **양 끝단 처리:** `range(1000, max_L, -1)`로 역순 순회 범위를 잡은 것도 아주 훌륭한 감각입니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)
로직의 흐름은 사용자님의 것과 100% 동일하지만, 불필요한 분기문을 `max()`로 걷어내어 가독성을 최고로 끌어올린 파이써닉 코드입니다.

```python
import sys
input = sys.stdin.readline

N = int(input())
storage = [0] * 1001

max_L, max_H = 0, 0

for _ in range(N):
    L, H = map(int, input().split())
    storage[L] = H
    # 가장 높은 기둥의 위치와 높이 찾기
    if H > max_H:
        max_H = H
        max_L = L

result = 0

# [왼쪽 파트] 0부터 max_L 직전까지
temp = 0
for i in range(max_L):
    # 빈 공간이든 아니든, 기존 최고 높이와 현재 높이 중 큰 쪽을 유지
    temp = max(temp, storage[i])
    result += temp

# [오른쪽 파트] 끝에서부터 max_L 직전까지 역순
temp = 0
for i in range(1000, max_L, -1):
    temp = max(temp, storage[i])
    result += temp

# [가장 높은 기둥 본체] 더하기
print(result + max_H)
```

💡 **최적화 핵심 원리:**

* **상태 변수와 `max()`의 조합:** 시뮬레이션에서 "현재까지 발견된 최댓값"을 갱신하며 무언가를 더해야 할 때는, 복잡한 `if`문 대신 `current_max = max(current_max, new_value)` 패턴을 사용하는 것이 오류를 줄이고 코드를 깔끔하게 만드는 파이썬의 절대 공식입니다.
