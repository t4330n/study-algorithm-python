### 1. 🔍 요구사항 분석 및 유형 파악

* **문제 유형:** 완전 탐색 (Brute-Force), 수학적 발상의 전환
* **유형 파악 키워드:** "아홉 명의 난쟁이 중 일곱 명", "합이 100" → 고정된 작은 크기의 집합에서 특정 조건(합)을 만족하는 조합을 찾는 전형적인 완전 탐색 문제입니다.
* **핵심 목표:** 9명 중 진짜 7명을 찾아 오름차순으로 출력하기.
* **입력 범위 및 시간 복잡도:** $N=9$로 매우 작습니다. 시간 제한이 2초이므로 어떤 비효율적인 탐색을 하더라도 무조건 통과하는, 시간 복잡도 제약이 사실상 없는 문제입니다.
* **주의할 엣지 케이스:** 반드시 오름차순으로 출력해야 합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

9명 중 7명을 찾는 방법도 있지만, 발상을 살짝 바꾸면 코드가 압도적으로 단순해집니다. **"9명 전체의 합에서 100을 빼면, 가짜 난쟁이 2명의 키의 합이 나온다"** 는 점을 이용하는 것입니다.

```markdown
1. 9명의 난쟁이 키를 입력받고 오름차순으로 정렬한다.
2. 9명 전체 키의 합을 구한 뒤, 100을 빼서 '가짜 난쟁이 2명의 키의 합(target)'을 구한다.
3. 9명 중 2명을 뽑는 모든 경우의 수를 확인한다. (2중 for문 또는 combinations)
4. 뽑힌 2명의 키의 합이 target과 같다면, 그 두 명이 가짜다!
5. 가짜 2명을 리스트에서 제거하고, 나머지 진짜 7명을 출력한 뒤 종료한다.
```

### 3. 💻 코드 구현 리뷰

작성하신 코드는 논리적으로 완벽하지만, 파이썬 환경에서는 **7중 `for`문**을 작성하는 것은 지양해야 할 패턴입니다.

* 이전 문제에서 배운 `itertools.combinations`를 사용했다면 `for comb in combinations(lengths, 7):` 단 두 줄로 7중 `for`문을 대체할 수 있었습니다.
* 또한, 7개의 변수(`a`~`g`)를 일일이 더하고 `append`하는 과정은 오타가 발생하기 쉽고 유지보수가 어렵습니다. 리스트 슬라이싱이나 내장 함수 `sum()`을 활용하는 습관을 들이면 좋습니다.

### 4. 🛡️ 엣지 케이스 점검

* 처음에 입력을 리스트 컴프리헨션과 `sorted()`를 통해 받아둔 덕분에, 어떤 순서로 정답을 찾든 오름차순 출력 조건이 완벽하게 방어되었습니다.
* 정답이 여러 개일 수 있는 경우(이 문제에서는 유일하다고 했지만), `solve()` 함수 내의 `return` 덕분에 첫 번째 정답을 찾자마자 깔끔하게 종료되어 불필요한 추가 출력을 막아냈습니다. 매우 훌륭한
  예외 처리입니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

앞서 설계한 **'가짜 난쟁이 2명 찾기'** 로직에 파이썬의 내장 함수를 곁들이면, 코드는 이렇게 경이로울 정도로 짧아집니다.

```python
import sys
from itertools import combinations

# 1. 입력 및 정렬
input = sys.stdin.readline
dwarfs = sorted([int(input()) for _ in range(9)])

# 2. 가짜 2명의 키의 합 계산
fake_sum = sum(dwarfs) - 100

# 3. combinations를 활용해 가짜 2명 찾기 (9명 중 2명 뽑기)
for f1, f2 in combinations(dwarfs, 2):
    if f1 + f2 == fake_sum:
        # 4. 가짜 2명을 리스트에서 제거
        dwarfs.remove(f1)
        dwarfs.remove(f2)
        break  # 정답을 찾았으므로 즉시 종료

# 5. 결과 출력
for dwarf in dwarfs:
    print(dwarf)
```

**💡 최적화 핵심 원리:**

1. **발상의 전환 ($^9C_7$ → $^9C_2$):** 7명을 찾는 것과 2명을 찾아 지우는 것은 결과적으로 같지만, 코드로 구현할 때는 탐색 깊이가 완전히 달라집니다.
2. **`remove()` 활용:** 리스트에서 특정 값을 바로 지워버리는 내장 함수를 사용해, 새로운 리스트에 7명을 다시 담는 수고를 덜었습니다.
