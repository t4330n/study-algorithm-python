### 1. 🔍 요구사항 분석 및 유형 파악

* **문제 유형:** 2차원 배열 (2D Array), 시뮬레이션, 상태 덮어쓰기
* **유형 파악 키워드:** "순서대로 놓인다", "보이지 않게 된다", "각 색종이가 보이는 부분의 면적" (순서가 매우 중요하며, 나중에 칠한 것이 이전 것을 덮어버린다는 것이 핵심 단서입니다.)
* **핵심 목표:** 복잡한 기하학적 교집합 계산을 피하고, 2차원 배열에 인덱스(1~N)를 직접 기록하여 최종적으로 남은 고유 번호의 개수를 세기.
* **입력 범위 및 시간 복잡도:** 평면 크기는 $1001 \times 1001$이며, 색종이는 최대 100장입니다. 만약 100장이 평면 전체를 덮는다면 배열을 순회하는 횟수가 급증하므로, 탐색 횟수를 최소화하는 전략이 필요합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

작성해주신 완벽한 로직의 흐름을 실전 코딩테스트용 뼈대로 정리해 드립니다.

```markdown
1. 1001 x 1001 크기의 2차원 평면 배열을 0으로 초기화한다.
2. 색종이의 개수 N을 입력받는다.
3. N번 반복하며 색종이의 좌표와 너비, 높이를 입력받는다.
4. 각 색종이의 영역을 2중 for문으로 순회하며, 해당 칸을 '현재 색종이의 번호(i)'로 덮어씌운다.
5. 모든 색칠이 끝나면 평면을 순회하며 1번부터 N번까지 각 숫자가 몇 개씩 살아남았는지 센다.
6. 세어진 면적을 순서대로 출력한다.
```

### 3. 💻 코드 구현 리뷰 (TLE 주의 구간)

색종이 번호를 마킹하는 `arr[x][y] = i` 접근법은 군더더기 없이 깔끔합니다. 하지만, 코딩테스트 플랫폼(특히 백준)에서 파이썬으로 제출할 때 마지막 면적을 구하는 부분에서 **시간 초과(또는 부분점수 53점)** 판정을 받을 위험이 큽니다.

* **비효율적인 3중 for문 구조:**
    ```python
    for i in range(1, N+1):          # 100번 반복
        for row in arr:              # 1001번 반복
            for col in row:          # 1001번 반복
    ```
    이 로직은 최악의 경우 $100 \times 1001 \times 1001 \approx 1억$ 번의 연산을 수행합니다. 타 언어에서는 0.1초 만에 통과되겠지만, 인터프리터 언어인 파이썬에서는 1초 제한을 넘길 아슬아슬한 수치입니다.

### 4. 🛡️ 엣지 케이스 점검

* **완전히 가려진 색종이:** 만약 1번 색종이 위에 2번 색종이가 똑같은 크기로 덮인다면? 1번 색종이의 면적은 0이 나와야 합니다. 작성하신 코드는 값을 덮어씌우는 구조이므로, 이 예외 상황을 완벽하게 방어해 내고 있습니다.
* **좌표와 인덱스의 매핑:** `(s_y, s_x)` 로 입력을 받아 `arr[x][y]` 형태로 직관적으로 뒤집어 사용하신 점이 매우 좋습니다. 2차원 배열에서 행(row)과 열(col)을 헷갈리지 않게 다루는 훌륭한 감각입니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

파이썬의 강력한 내장 기능과 리스트 슬라이싱을 활용하여, 초고속 최적화 코드를 보여드립니다. 이 기법들은 노션에 '파이썬 2차원 배열 최적화' 키워드로 아카이빙해 두시면 매우 유용합니다!

```python
import sys
input = sys.stdin.readline

arr = [[0] * 1001 for _ in range(1001)]
N = int(input())

for i in range(1, N + 1):
    s_y, s_x, w, h = map(int, input().split())
    
    # [최적화 1] 2중 for문 대신, 리스트 슬라이싱을 이용한 초고속 색칠
    for x in range(s_x, s_x + h):
        # x행의 s_y부터 s_y+w-1까지의 구간을 통째로 [i] 배열로 덮어씌움
        arr[x][s_y : s_y + w] = [i] * w

# [최적화 2] 3중 for문 대신, count() 내장 함수를 이용한 초고속 개수 세기
for i in range(1, N + 1):
    count = 0
    for row in arr:
        # 각 행(1차원 리스트)에서 i의 개수를 세어 더함 (C언어로 구현되어 있어 압도적으로 빠름)
        count += row.count(i)
    print(count)
```

💡 **최적화 핵심 원리:**

1.  **리스트 슬라이싱 (`arr[x][s_y : s_y+w] = [i] * w`)**: 파이썬에서 하나하나의 원소를 for문으로 바꾸는 것은 오버헤드가 큽니다. 슬라이싱을 이용해 배열의 특정 구간을 한 번에 갈아 끼우면 속도가 비약적으로 상승합니다.
2.  **`row.count(i)`의 활용**: 파이썬의 `.count()` 내장 함수는 내부적으로 C언어로 최적화되어 동작합니다. 직접 for문으로 요소를 비교하며 세는 것보다 수십 배 빠르게 동작하여 시간 초과를 완벽하게 방어합니다.
