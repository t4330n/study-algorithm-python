### 1. 🔍 요구사항 분석 및 유형 파악

* **문제 유형:** 1차원 배열 시뮬레이션, 구현 (Implementation)
* **유형 파악 키워드:** "가장 많은 조각을 받을 것으로 기대", "실제로 가장 많이 받은 방청객", "동일할 경우 번호가 작은 사람을 출력"
* **핵심 목표:** 1차원 배열을 빈 롤 케이크(0)로 두고, 방청객 순서대로 배열을 탐색하며 아직 주인이 없는 조각(0)을 자신의 번호로 칠해나가기.
* **입력 범위 및 시간 복잡도:** 롤 케이크 길이 $L$은 최대 1000, 방청객 수 $N$도 최대 1000입니다. 2중 for문으로 전체를 시뮬레이션하더라도 최악의 경우 약 100만 번($1000 \times 1000$)의 연산만 수행하므로 1초 제한을 아주 여유롭게 통과합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

작성해주신 코드의 흐름을 가장 이상적인 실전 뼈대로 다듬어 보았습니다.

```markdown
1. 롤 케이크의 길이 L을 입력받고, (L+1) 크기의 1차원 배열을 0으로 초기화한다.
2. 방청객의 수 N을 입력받는다.
3. 기댓값이 가장 큰 사람과 실곗값이 가장 큰 사람을 추적할 변수를 초기화한다.
4. 1번 방청객부터 N번 방청객까지 순서대로 (시작점 p, 끝점 k)를 입력받는다.
5. [기댓값 갱신] (끝점 - 시작점 + 1)을 계산하여, 기존 최대 기댓값보다 크면 번호를 갱신한다.
6. [실곗값 갱신] 시작점부터 끝점까지 롤 케이크 배열을 순회하며,
7. 해당 조각이 0(빈 조각)이라면 현재 방청객 번호로 칠하고 카운트를 올린다.
8. 이 방청객이 얻은 총 카운트가 기존 최대 실곗값보다 크면 번호를 갱신한다.
```

### 3. 💻 코드 구현 리뷰 (숨은 논리적 버그 발견!)

로직은 완벽하게 정답을 도출해 냈지만, 짚고 넘어가야 할 **개념적 버그(Off-by-one Error)** 가 하나 숨어 있습니다. 

* **조각 개수 계산의 함정:**
  사용자님은 기대하는 조각의 수를 `k - p`로 계산하셨습니다. 하지만 만약 1번 조각부터 3번 조각까지 원한다면 `3 - 1 = 2`가 아니라, 1번, 2번, 3번으로 **총 3조각**(`k - p + 1`)입니다.
  모든 방청객의 조각 수 계산에서 똑같이 1씩 모자라게 계산되었기 때문에 크기 비교(`>`)에서는 우연히 대소 관계가 유지되어 정답 처리를 받을 수 있었습니다. 하지만 만약 문제에서 '그 사람이 기대했던 조각의 수'를 같이 출력하라고 했다면 오답이 되었을 것입니다. 길이나 개수를 셀 때는 항상 **`+1`** 을 잊지 않도록 주의해야 합니다!

### 4. 🛡️ 엣지 케이스 점검

* **동점자 발생 처리 (가장 중요):** 가장 많은 케이크를 받은 사람이 여러 명일 때, 번호가 작은 사람을 출력해야 합니다. 
  사용자님의 코드 `if count > real[1]:` 는 개수가 완벽히 클 때만(`>`) 최대치를 갱신하므로, 나중에 똑같은 개수를 받은 뒷번호 사람이 나타나도 무시하게 됩니다. 이 문제의 가장 까다로운 함정을 완벽하게 방어해 낸 최고의 한 수입니다!

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

리뷰 내용을 바탕으로, 조각 개수 계산(`k - p + 1`)을 올바르게 수정하고, 배열 탐색 시 `!= 0`일 때 `continue`로 건너뛰는 방식 대신 `== 0`일 때만 색칠하는 긍정문으로 바꾸어 가독성을 높인 파이써닉 코드입니다.

```python
import sys
input = sys.stdin.readline

L = int(input())
N = int(input())

roll_cake = [0] * (L + 1)

# [번호, 최대 개수]를 저장할 변수
max_expect = [0, 0]
max_real = [0, 0]

for i in range(1, N + 1):
    p, k = map(int, input().split())
    
    # 1. 기댓값 계산 및 갱신 (반드시 +1을 해주어야 실제 조각 개수가 됩니다!)
    expected_count = k - p + 1
    if expected_count > max_expect[1]:
        max_expect = [i, expected_count]
        
    # 2. 실제 얻는 조각 계산 및 배열 갱신
    real_count = 0
    for idx in range(p, k + 1):
        # 비어있는(0) 케이크만 내 번호(i)로 칠하기
        if roll_cake[idx] == 0:
            roll_cake[idx] = i
            real_count += 1
            
    # 3. 실곗값 갱신
    if real_count > max_real[1]:
        max_real = [i, real_count]

print(max_expect[0])
print(max_real[0])
```

💡 **최적화 핵심 원리:**

코드의 뼈대는 사용자님의 것과 거의 같습니다. 그만큼 처음에 설계하신 아이디어가 코딩테스트 시뮬레이션의 정석에 가깝다는 뜻입니다! 
다만 `expect[1]`과 `real[1]`에 접근하는 대신, 명시적인 변수(`expected_count`, `real_count`)에 먼저 값을 담고 비교하면 디버깅이 훨씬 쉬워지고 코드를 읽는 속도도 빨라집니다.
