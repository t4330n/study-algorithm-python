### 1. 🔍 요구사항 분석 및 유형 파악 (Upgrade!)

* **문제 유형:** 동적 계획법 (Dynamic Programming, DP)
* **유형 파악 키워드:** "연산을 사용하는 횟수의 **최솟값**", "$N$은 $10^6$ 이하이다"
    * $N$이 주어졌을 때 이전의 결과(작은 숫자를 1로 만드는 횟수)를 재활용하여 현재의 최솟값을 찾아내는 완벽한 점화식 구조입니다.
* **핵심 목표:** `dp[i]` = $i$를 1로 만드는 데 필요한 최소 연산 횟수 구하기.
* **입력 범위 및 시간 복잡도:** $N \le 1,000,000$. 피보나치처럼 1부터 $N$까지 배열을 채워 올라가는 $O(N)$ 시간 복잡도로 0.15초(파이썬은 약간 더 여유 있음) 안에 통과해야 합니다.
* **주의할 엣지 케이스:** $N=1$일 때는 아무 연산도 필요 없으므로 답이 0이어야 합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

피보나치 수열은 무조건 `f(n-1) + f(n-2)` 였지만, 이 문제는 **3가지 선택지 중 가장 작은 값**을 골라야 합니다.

```markdown
1. 정수 N을 입력받는다.
2. 연산 횟수를 저장할 DP 배열을 N+1 크기로 만들고 0으로 초기화한다.
   (dp[1]은 어차피 0이므로, 2부터 시작하면 된다)
3. 2부터 N까지 반복문을 돌며 점화식을 세운다:
   3-1. [기본] 현재 수(i)에서 1을 빼는 경우: dp[i] = dp[i-1] + 1
   3-2. [비교] 만약 i가 2로 나누어 떨어진다면:
   1을 뺀 결과(dp[i])와 2로 나눈 결과(dp[i//2] + 1) 중 더 작은 값으로 갱신!
   3-3. [비교] 만약 i가 3으로 나누어 떨어진다면:
   지금까지의 최솟값(dp[i])과 3으로 나눈 결과(dp[i//3] + 1) 중 더 작은 값으로 갱신!
4. 최종적으로 완성된 dp[N]을 출력한다.
```

### 3. 💻 코드 구현 (Pythonic)

뼈대대로 구현한 가장 정석적이고 직관적인 바텀업 DP 코드입니다.

```python
import sys

input = sys.stdin.readline

N = int(input())

# 1. DP 배열 생성 (인덱스를 직관적으로 쓰기 위해 N+1 크기로)
dp = [0] * (N + 1)

# 2. 2부터 N까지 순회하며 DP 배열 채우기
for i in range(2, N + 1):
    # 기본적으로 1을 빼는 연산을 수행한다고 가정 (+1회)
    dp[i] = dp[i - 1] + 1

    # 현재 수가 2로 나누어 떨어진다면
    if i % 2 == 0:
        # '1을 뺐을 때의 횟수'와 '2로 나누었을 때의 횟수 + 1' 중 최솟값 갱신
        dp[i] = min(dp[i], dp[i // 2] + 1)

    # 현재 수가 3으로 나누어 떨어진다면
    if i % 3 == 0:
        # '지금까지 구한 최솟값'과 '3으로 나누었을 때의 횟수 + 1' 중 최솟값 갱신
        dp[i] = min(dp[i], dp[i // 3] + 1)

print(dp[N])
```

### 4. 🛡️ 엣지 케이스 점검

* **그리디의 함정 ($N=10$):**
    * 무조건 큰 수로 나누는 것이 이득이라고 착각하여 10 $\rightarrow$ 5 $\rightarrow$ 4 $\rightarrow$ 2 $\rightarrow$ 1 (4번)을 도출하기 쉽습니다.
    * 하지만 DP는 `i=10`일 때, 1을 먼저 빼는 루트인 `dp[9] + 1`을 검사합니다. 9는 3으로 두 번 나누어 떨어지므로 10 $\rightarrow$ 9 $\rightarrow$
      3 $\rightarrow$ 1 (3번)이라는 진정한 최적해를 완벽하게 찾아냅니다.
* **$N=1$인 경우:** `range(2, 2)`가 되어 for문이 아예 실행되지 않고 초기에 설정한 `dp[1] = 0`이 안전하게 출력됩니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

바텀업 방식은 1부터 $N$까지 모든 숫자의 결과를 다 계산해야 하므로 무조건 $O(N)$의 시간이 걸립니다.
하지만 이 문제를 파이썬의 **딕셔너리를 활용한 재귀(Top-down DP)** 로 풀면, 필요 없는 중간 숫자의 계산을 전부 건너뛰어 속도를 기존 대비 **수십 배 이상** 폭발적으로 끌어올릴 수 있습니다.

```python
N = int(input())

# 메모이제이션을 위한 딕셔너리 (초기값 세팅)
memo = {1: 0, 2: 1}


def solve(n):
    # 이미 계산한 적 있는 숫자라면 딕셔너리에서 바로 꺼내 씀
    if n in memo:
        return memo[n]

    # 핵심 원리: 1씩 무식하게 빼는 대신, 
    # '3으로 나눈 나머지'나 '2로 나눈 나머지'만큼 한 번에 빼버린 뒤 나눗셈 진행
    # n // 3 으로 갔을 때의 횟수와 n // 2 로 갔을 때의 횟수 중 최솟값 찾기
    memo[n] = min(solve(n // 3) + n % 3 + 1, solve(n // 2) + n % 2 + 1)

    return memo[n]


print(solve(N))
```

💡 **최적화 핵심 원리:**

$N=1,000,000$일 때, 1번 코드는 배열을 100만 번 순회합니다. 반면 이 최적화 코드는 숫자를 3과 2로 큼직하게 덩어리째 쪼개면서 내려가기 때문에 탐색 깊이가 $O(\log N)$에 가깝게 줄어들어,
단 몇십 번의 재귀 호출만으로 답을 도출해냅니다.
