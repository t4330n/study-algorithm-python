### 1. 🔍 요구사항 분석 및 유형 파악

* **문제 유형:** 2차원 배열 시뮬레이션 (2D Array Simulation), 델타 탐색 (상하좌우 이동)
* **유형 파악 키워드:** 
    * "반시계 방향으로 돌아가면서", "K번 관객에게 배정될 좌석 번호"
    * 2차원 격자에서 특정 방향으로 직진하다가 벽이나 장애물을 만나면 90도 꺾는 전형적인 **나선형(Spiral) 탐색** 문제입니다.
* **핵심 목표:** 방향 배열(`dx`, `dy`)을 정의하고, 격자를 벗어나거나 이미 방문한 칸을 만나면 방향을 전환하며 $K$번째 방문 좌표를 찾기.
* **입력 범위 및 시간 복잡도:** 가로 $C$, 세로 $R$은 최대 1,000입니다. 전체 좌석 수는 최대 1,000,000개이므로 배열을 전부 탐색해도 $O(CR)$ 알고리즘으로 1초 안에 충분히 통과합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

달팽이 배열의 가장 이상적인 실전 뼈대를 정리해 드립니다. 

```markdown
1. C(가로)와 R(세로), 그리고 찾고자 하는 번호 K를 입력받는다.
2. K가 C * R 보다 크면 좌석을 줄 수 없으므로 0을 출력하고 즉시 종료한다.
3. 방문 여부를 체크할 2차원 배열(visited)을 만든다.
4. 방향 벡터(상, 우, 하, 좌)를 순서대로 정의한다. (문제에서는 위로 먼저 출발하므로 y축 증가 방향이 1번)
5. 1번 좌석부터 K번 좌석까지 반복문을 돌며 현재 위치에 방문 표시를 한다.
6. '다음 좌표'가 격자를 벗어나거나 이미 방문한 곳이라면, 방향 인덱스를 +1 하여 90도 회전시킨다.
7. K번째 좌석에 도달하면 해당 좌표를 출력하고 탐색을 곧바로 종료한다.
```

### 3. 💻 코드 구현 리뷰 (비효율적인 반복문 발견!)

델타 탐색 배열(`di`)과 `dc%4`를 사용한 논리는 100점 만점에 100점입니다! 하지만, 코딩테스트 관점에서 **치명적인 시간/메모리 낭비 구간**이 존재합니다.

* **불필요한 전체 탐색과 후위 검색:**
    사용자님의 코드는 $K=1$이 주어지더라도, 무조건 $1,000 \times 1,000$ (100만 번) 루프를 돌아 배열을 끝까지 꽉 채웁니다. 그다음 또다시 2중 for문을 돌며 $K$의 위치를 찾고 있습니다. 
    우리의 목표는 $K$번째 관객의 자리만 알면 되므로, **반복문을 $K$번까지만 돌고 그 순간의 x, y 좌표를 출력한 뒤 프로그램을 아예 종료해버리면** 실행 시간을 엄청나게 단축할 수 있습니다.

### 4. 🛡️ 엣지 케이스 점검

* **대기번호가 좌석 수보다 많은 경우:** `if K > row * col: print(0)` 구문으로 가장 핵심적인 예외 처리를 초반에 완벽하게 방어해 내셨습니다. 이 조건문 덕분에 배열 범위 초과 에러(IndexError) 위험이 완전히 사라졌습니다.
* **배열 패딩(Padding):** `arr = [[0] * (col+2) for _ in range(row+2)]` 로 배열 주변에 여유 공간을 주어 테두리 인덱스 에러를 방지한 감각도 매우 훌륭합니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

리뷰 내용을 반영하여, $K$번째 도달 즉시 탐색을 멈추어(조기 종료) 연산 횟수를 획기적으로 줄이고, 코드를 더 간결하게 다듬은 파이써닉 시뮬레이션 코드입니다.

```python
import sys
input = sys.stdin.readline

C, R = map(int, input().split())
K = int(input())

# [최적화 1] 애초에 좌석을 배정할 수 없다면 0 출력 후 프로그램 종료
if K > C * R:
    print(0)
    exit()

arr = [[0] * (R + 1) for _ in range(C + 1)]

# 문제의 조건에 맞게: 상(0,1), 우(1,0), 하(0,-1), 좌(-1,0)
dx = [0, 1, 0, -1]
dy = [1, 0, -1, 0]

x, y = 1, 1
direction = 0

# [최적화 2] 전체(C*R)를 다 돌지 않고, 정확히 K번까지만 시뮬레이션
for i in range(1, K + 1):
    # K번째 좌석에 도달했다면 바로 좌표 출력 후 종료 (2중 for문 검색 방지)
    if i == K:
        print(x, y)
        break
        
    arr[x][y] = i # 현재 위치 방문 처리
    
    # 다음 이동할 위치 계산
    nx = x + dx[direction]
    ny = y + dy[direction]
    
    # 다음 위치가 범위를 벗어나거나 이미 방문한 곳(0이 아님)이라면 방향 전환
    if nx < 1 or nx > C or ny < 1 or ny > R or arr[nx][ny] != 0:
        direction = (direction + 1) % 4
        nx = x + dx[direction]
        ny = y + dy[direction]
        
    # 좌표 업데이트
    x, y = nx, ny
```

💡 **최적화 핵심 원리:**

**"필요한 만큼만 연산하라(조기 종료)"** 는 백트래킹과 시뮬레이션의 핵심입니다. 
$K$를 찾기 위해 전체 배열을 다 그리는 것은 '비효율적인 완탐'에 속합니다. 우리가 일상생활에서 원하는 물건을 찾으면 방 뒤지기를 멈추듯, 코딩테스트에서도 정답을 찾은 순간 `break`나 `exit()`로 로직을 칼같이 끊어내야 시간 초과(TLE)의 늪에 빠지지 않습니다.
