### 1. 🔍 요구사항 분석 및 유형 파악 (Upgrade!)

* **문제 유형:** 2차원 좌표 시뮬레이션 (2D Coordinate Simulation)
* **유형 파악 키워드:** "알파벳은 열, 숫자는 행", "체스판 밖으로 나갈 경우 그 이동은 건너뛴다", "돌과 같은 곳으로 이동할 때"
    * 배열의 인덱스가 아닌, 수학 시간의 **'데카르트 좌표계(x, y축)'** 로 접근해야 하는 문제입니다.
* **핵심 목표:** 킹과 돌의 (x, y) 좌표를 이동시키되, 이동하려는 곳에 돌이 있으면 돌도 같이 밀어내기. 단, 둘 중 하나라도 체스판 밖으로 나가면 해당 턴 자체를 무효화하기.
* **입력 범위 및 시간 복잡도:** 이동 횟수 $N \le 50$. N이 매우 작으므로 시간 복잡도는 전혀 고려할 필요가 없는 $O(N)$ 단순 구현 문제입니다.
* **주의할 엣지 케이스:** **(사용자님 코드의 가장 큰 오답 원인)** 킹이 돌을 밀어냈는데, 그 밀려난 돌이 체스판 밖으로 나가는 경우입니다. 이때는 **킹의 이동마저도 취소(건너뜀)** 되어야 합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

굳이 8x8 2차원 배열(맵)을 메모리에 만들 필요가 **전혀 없습니다.** 킹과 돌의 현재 x, y 좌표값만 변수로 들고 있으면 됩니다.

```markdown
1. 킹 위치, 돌 위치, 이동 횟수 N을 입력받는다.
2. 문자로 된 위치(예: A1)를 다루기 쉬운 숫자 좌표(0~7, 0~7)로 변환한다.
3. 8가지 방향에 대한 (dx, dy) 딕셔너리를 정의한다. (이때 y축은 위로 갈수록 증가함!)
4. N번 반복하며 이동 명령을 처리한다:
   4-1. 킹의 '다음 위치'를 계산한다.
   4-2. 킹의 다음 위치가 체스판(0~7) 밖이라면 건너뛴다(continue).
   4-3. 킹의 다음 위치에 '돌'이 있다면:
   돌의 '다음 위치'도 계산한다.
   돌의 다음 위치가 체스판 밖이라면 건너뛴다(continue). -> 이동 무효화!
   돌의 위치를 갱신한다.
   4-4. 킹의 위치를 갱신한다.
5. 숫자 좌표를 다시 문자열(A1 형태)로 변환하여 출력한다.
```

### 3. 💻 코드 구현 (Pythonic)

사용자님이 길게 작성하셨던 `col` 딕셔너리와 타입 변환 과정을 파이썬의 `ord()` 함수로 한 줄에 끝내고, 맵 없이 좌표만으로 이동하는 직관적인 코드입니다.

```python
import sys

input = sys.stdin.readline

k_pos, s_pos, N = input().split()

# 문자를 0~7 사이의 좌표로 변환 (A->0, 1->0)
# ord('A')는 65이므로, 문자에 ord()를 씌우고 65를 빼면 깔끔한 인덱스가 됩니다.
kx, ky = ord(k_pos[0]) - 65, int(k_pos[1]) - 1
sx, sy = ord(s_pos[0]) - 65, int(s_pos[1]) - 1

# 수학 좌표계처럼 오른쪽(x증가), 위(y증가)로 세팅
moves = {
    "R": (1, 0), "L": (-1, 0), "B": (0, -1), "T": (0, 1),
    "RT": (1, 1), "LT": (-1, 1), "RB": (1, -1), "LB": (-1, -1)
}

for _ in range(int(N)):
    m = input().strip()
    dx, dy = moves[m]

    nkx, nky = kx + dx, ky + dy

    # 1. 킹이 체스판 밖으로 나가는지 확인
    if not (0 <= nkx < 8 and 0 <= nky < 8):
        continue

    # 2. 킹이 이동하려는 곳에 돌이 있는 경우
    if nkx == sx and nky == sy:
        nsx, nsy = sx + dx, sy + dy
        # 3. 밀려난 돌이 체스판 밖으로 나가는지 확인 (핵심 방어선)
        if not (0 <= nsx < 8 and 0 <= nsy < 8):
            continue
        # 돌 위치 갱신
        sx, sy = nsx, nsy

    # 4. 킹 위치 갱신
    kx, ky = nkx, nky

# 숫자를 다시 문자로 변환 (0->A, 0->1)
print(f"{chr(kx + 65)}{ky + 1}")
print(f"{chr(sx + 65)}{sy + 1}")
```

### 4. 🛡️ 엣지 케이스 점검

* **배열 뒤집기 문제 해결:** 문제에서 1행이 가장 아래, 8행이 가장 위라고 했습니다. 이를 2차원 배열 인덱스로 맞추려니 8에서 빼고 더하고 복잡해졌죠. 하지만 **그냥 수학 좌표계라고 생각하고 `T(위)`
  로 갈 때 `y+1`을 해버리면** 상하 반전을 고민할 필요가 아예 사라집니다. 출력할 때도 그대로 쓰면 되고요!
* **돌이 밀려나는 예외 처리:** 사용자님의 코드에서는 킹과 돌이 겹쳤을 때 돌을 무조건 이동시켜버린 후 킹의 유효성을 검사했습니다. 만약 돌이 체스판 밖으로 나갔다면 턴이 무효화되어 킹도 돌도 원래 자리로
  돌아가야 하는데, 이미 값이 덮어씌워져 버그가 발생합니다. 최적화된 코드에서는 `nsx, nsy`로 먼저 확인한 뒤 안전할 때만 덮어씌우도록 방어했습니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

위 코드가 곧 시뮬레이션의 정석이자 최적화 코드입니다. 이 문제에서 얻어가셔야 할 가장 중요한 핵심 원리는 다음과 같습니다.

💡 **최적화 핵심 원리:**

1. **아스키코드 변환 (`ord`, `chr`):** 알파벳 좌표(A~H, a~z 등)가 주어지는 맵 이동 문제에서는 알파벳을 하드코딩된 딕셔너리로 매핑하지 않습니다. `ord('문자') - ord('A')`를
   사용하면 'A'는 0, 'B'는 1로 자동 변환되며, 반대로 출력할 때는 `chr(숫자 + 65)`로 단숨에 복구할 수 있습니다.
2. **"맵(Map)리스" 시뮬레이션:** 모든 2차원 공간 문제가 2차원 리스트(`[][]`)를 요구하는 것은 아닙니다. 장애물이나 길의 상태가 계속 변하는 미로 찾기가 아니라면, 이 체스판 문제처럼 **'객체의
   x, y 좌표값'** 만 들고 수학적인 덧셈 뺄셈만 하는 것이 메모리도 적게 먹고 인덱스 에러에서도 자유로운 가장 파이써닉한 풀이법입니다.
