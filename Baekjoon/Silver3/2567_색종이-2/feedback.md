### 1. 🔍 요구사항 분석 및 유형 파악

* **문제 유형:** 2차원 배열 시뮬레이션, 경계선 탐색 (DFS/BFS의 기초)
* **유형 파악 키워드:** "둘레의 길이"
* **핵심 목표:** 도화지 위에서 겹쳐진 색종이들의 전체 외곽선 길이를 구하기.
* **입력 범위 및 시간 복잡도:** 102x102 배열을 순회하며 4방향을 탐색하므로 약 4만 번의 연산으로 충분합니다. $O(N^2)$ 완전 탐색이 가장 확실한 정답입니다.
* **주의할 엣지 케이스:** 도화지 경계(x=0 또는 y=0)에 색종이가 딱 붙어 있는 경우 경계선을 정확히 인식해야 합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

가장 직관적이고 예외가 없는 '패딩(Padding)' 기법의 뼈대입니다.

```markdown
1. 상하좌우 경계선 바깥을 0으로 둘러싸기 위해 102x102 크기의 배열을 0으로 초기화한다.
2. 색종이 좌표를 입력받을 때, 무조건 +1을 더해 '패딩 안쪽'에 색종이가 칠해지도록 한다.
3. 색종이 크기(10x10)만큼 배열의 값을 1로 바꾼다.
4. 102x102 배열 전체를 순회하며 1(색종이)을 찾는다.
5. 1의 상하좌우를 탐색하여 0(빈 공간)이 있으면 그곳이 테두리이므로 둘레 길이를 +1 한다.
6. 최종 합산된 둘레 길이를 출력한다.
```

### 3. 💻 코드 구현 리뷰

✨ **칭찬할 점:**

* 이전에 실수하셨던 `+ 11`을 `+ 10`으로 완벽하게 수정하셨습니다!
* `direction = [[-1, 0], [1, 0], [0, 1], [0, -1]]` 배열을 활용한 델타 탐색은 2차원 배열 시뮬레이션에서 사용하는 아주 정석적이고 훌륭한 패턴입니다.

💣 **숨은 폭탄 (백준 테케의 맹점):**
작성하신 코드에서 `[[0] + [0] * 100 + [0] ...]` 형태로 패딩을 의도하신 것은 매우 훌륭합니다. 하지만 만약 문제의 입력으로 도화지 왼쪽 아래 끝점인 **`0 0`** 이 들어왔다면 어떻게
될까요?

* `arr[0][0]`부터 `arr[9][9]`까지 1로 칠해집니다.
* 의도하셨던 0번 인덱스의 '0 패딩'이 1로 덮어씌워져 버립니다!
* 이후 `for x in range(102)`를 돌며 0을 찾을 때, `x=0, y=0` 주변의 바깥쪽 경계(예: `x=-1` 위치)를 감지해 줄 0이 배열에 존재하지 않게 됩니다.
* 결국 왼쪽과 아래쪽 모서리의 둘레 길이가 카운트되지 않고 누락됩니다.

백준의 테스트 케이스에 x나 y가 0에 바짝 붙는 극단적인 예외가 없어서 '맞았습니다' 처리가 되었을 뿐, 실전 SSAFY 코딩테스트였다면 엣지 케이스에서 히든 테케 실패가 뜰 수 있었습니다.

### 4. 🛡️ 엣지 케이스 점검

이 폭탄을 완전히 제거하고 '경계선 누락' 엣지 케이스를 완벽히 방어하는 유일한 방법은, **색종이를 칠할 때 입력받은 x, y 좌표에 무조건 + 1을 해서 배열 안쪽으로 한 칸 밀어 넣는 것(Shift)**
입니다.
이렇게 하면 색종이가 도화지 끝에 붙어도 인덱스 0번과 101번 라인은 영원히 0(패딩)으로 안전하게 남아있게 됩니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

사용자님의 델타 탐색 논리를 그대로 유지하되, 좌표 시프트(+1)를 적용하여 패딩을 무적 상태로 만들고, 0을 찾는 대신 "1(색종이)을 찾아서 주변의 0(바깥 공기)과 닿은 면을 세는" 방식으로 살짝 뒤집은 최적화
코드입니다.

```python
import sys

input = sys.stdin.readline
N = int(input())

# 1. 102x102 배열 생성 (0번과 101번 인덱스는 영구적인 패딩 역할)
arr = [[0] * 102 for _ in range(102)]

for _ in range(N):
    x, y = map(int, input().split())
    # 2. [최적화 핵심] 좌표에 +1을 해서 무조건 패딩 안쪽에 칠해지도록 강제
    for i in range(x + 1, x + 11):
        for j in range(y + 1, y + 11):
            arr[i][j] = 1

direction = [[-1, 0], [1, 0], [0, 1], [0, -1]]
perimeter = 0

# 3. 전체 도화지를 순회하며 1(색종이)을 찾음
for i in range(1, 101):
    for j in range(1, 101):
        if arr[i][j] == 1:
            for dx, dy in direction:
                nx, ny = i + dx, j + dy
                # 4. 내 주변에 0이 있다? -> 그곳이 바로 닿아있는 둘레(경계선)!
                if arr[nx][ny] == 0:
                    perimeter += 1

print(perimeter)
```

**💡 최적화 핵심 원리:**

1. **탐색의 주체 변경:** 사용자님의 코드는 "0을 찾아서 1을 세는" 방식이라 빈 공간(100x100) 전체를 돌며 연산이 잦을 수 있습니다. 반면, "1을 찾아서 0을 세는" 방식은 오직 색종이가 칠해진
   구역 안에서만 경계선을 확인하므로 논리적으로 더 자연스럽고, 내부 빈 공간(구멍)의 둘레까지 완벽하게 잡아냅니다.
2. **`x+1`, `y+1` 시프트:** 패딩이 파괴되는 것을 원천 차단했습니다. 배열의 인덱스 에러나 경계선 누락을 방어하는 최고의 테크닉입니다.
