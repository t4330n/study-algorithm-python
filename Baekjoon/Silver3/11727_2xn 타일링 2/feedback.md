### 1. 🔍 요구사항 분석 및 유형 파악 (Upgrade!)

* **문제 유형:** 동적 계획법 (Dynamic Programming, DP)
* **유형 파악 키워드:** "$2 \times n$ 직사각형을 채우는 방법의 수", "**10,007로 나눈 나머지**"
    * "방법의 수"를 구하는데 "나눈 나머지"를 요구한다면, 정답이 **기하급수적으로 커진다는 뜻**입니다. 이는 앞의 계산 결과를 계속 재활용해야 하는 전형적인 DP의 강력한 힌트입니다.
* **핵심 목표:** `dp[i] = dp[i-1] + dp[i-2] * 2` 라는 점화식을 도출하여 $n$번째 경우의 수 구하기.
* **입력 범위 및 시간 복잡도:** $n \le 1,000$. 반복문 한 번으로 끝나는 $O(N)$ 시간 복잡도면 충분합니다.
* **주의할 엣지 케이스:** $n=1$일 때 인덱스 에러가 발생하지 않도록 초기 배열 크기와 반복문 범위를 잘 조율해야 합니다.

### 2. 🧠 논리적 설계 (주석 뼈대)

경우의 수를 쪼개어 점화식을 세우는 DP 설계의 정석입니다.

```markdown
1. 2xN 크기를 채우는 방법은 크게 두 가지에서 파생된다.
2. [가장 마지막에 2x1 타일 1개를 세로로 두는 경우] -> (N-1) 크기를 채우는 방법의 수와 동일
3. [가장 마지막에 1x2 타일 2개를 가로로 두거나, 2x2 타일 1개를 두는 경우]
   -> (N-2) 크기를 채우는 방법에 2가지 경우의 수가 곱해짐
4. 따라서 점화식은 DP[i] = DP[i-1] + (DP[i-2] * 2) 가 된다.
5. 매 덧셈 연산마다 10007로 나눈 나머지를 저장하여 숫자가 너무 커지는 것을 방지한다.
```

### 3. 💻 코드 구현 리뷰 (치명적인 파이썬의 함정!)

사용자님의 코드는 배열을 크게 잡아서 인덱스 에러를 방지한 아주 좋은 코드입니다. 하지만 **파이썬의 '무한 정수 처리' 특성** 때문에 숨겨진 비효율이 하나 발생했습니다.

* **모듈러 연산(`% 10007`)의 위치:**
  사용자님은 반복문을 다 돌고 나서 마지막 출력 때 `lst[n] % 10007`을 하셨습니다. 타 언어였다면 정수 범위를 초과해서 오버플로우 에러가 났겠지만, 파이썬은 똑똑하게도 숫자가 아무리 커져도 메모리를
  무한정 써서 계산해 냅니다.
  하지만 $n=1000$쯤 되면 `lst[1000]`은 **수백 자리에 달하는 엄청나게 긴 숫자**가 됩니다. 이 거대한 숫자를 계속 더하다 보면 덧셈 연산 자체가 눈에 띄게 느려지고 메모리도 낭비됩니다. *
  *반드시 루프 안에서 매번 더할 때마다 `% 10007`을 해주어야** 값을 항상 10007 이하로 가볍게 유지할 수 있습니다.
* **불필요한 반복:** $n=5$가 들어와도 무조건 1000까지 루프를 도는 부분을 `range(3, n + 1)`로 줄이면 불필요한 연산을 없앨 수 있습니다.

### 4. 🛡️ 엣지 케이스 점검

* **$n=1$인 경우:** `lst[2] = 3`을 먼저 선언하셨지만, `lst = [0] * 1001`로 리스트 길이를 이미 1001로 넉넉하게 뚫어놓으셨기 때문에 IndexError 없이 안전하게
  넘어갔습니다. 아주 훌륭한 방어막이었습니다!

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

피보나치 때 적용해 보았던 **'공간 최적화 (변수 2개만 사용하기)'** 기법과 **'실시간 모듈러 연산'** 을 적용하여, 메모리 사용량을 $O(1)$로 줄이고 속도를 극대화한 최적화 코드입니다.

```python
import sys

input = sys.stdin.readline

n = int(input())

# 초기값 세팅: a는 1번째(1), b는 2번째(3) 경우의 수
a, b = 1, 3

# n이 1이거나 2일 때는 루프를 돌 필요 없이 바로 조건 처리
if n == 1:
    print(1)
elif n == 2:
    print(3)
else:
    # 3부터 n까지 반복
    for _ in range(3, n + 1):
        # 파이썬 다중 할당 및 실시간 모듈러 연산
        # 다음 a는 현재의 b가 되고, 다음 b는 (현재 a * 2 + 현재 b) % 10007 이 됨
        a, b = b, (a * 2 + b) % 10007

    print(b)
```

💡 **최적화 핵심 원리:**

이전의 피보나치 문제에서 배우셨던 `a, b = b, a + b` 구조를 그대로 가져와 점화식 부분만 수정한 것입니다.
DP 배열 전체를 `lst[i]`로 유지할 필요 없이, **'직전 값(`b`)'** 과 **'두 칸 전 값(`a`)'** 만 계속 갱신하며 나아가면 메모리를 극한으로 아낄 수 있습니다. 여기에 매번 `% 10007`
을
해줌으로써 숫자가 수백 자리로 폭주하는 현상을 완벽히 차단했습니다.
