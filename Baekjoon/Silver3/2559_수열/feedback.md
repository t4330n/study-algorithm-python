### 1. 🔍 요구사항 분석 및 유형 파악 (Upgrade!)

* **문제 유형:** 슬라이딩 윈도우 (Sliding Window), 누적 합 (Prefix Sum)
* **유형 파악 키워드 (가장 중요!):**
    1. **"연속적인 며칠 동안($K$)의 온도의 합"**: 구간의 길이($K$)가 고정되어 있으며, 값들이 끊어지지 않고 이어져 있습니다. (창틀의 크기가 고정된 채로 이동하는 모습)
    2. **"$N$은 최대 100,000"**: 만약 2중 for문으로 매번 $K$개를 새로 더한다면 연산량이 $O(N \times K)$가 되어 최악의
       경우 $100,000 \times 100,000 = 100억$ 번의 연산이 발생합니다. 파이썬은 1초(약 2천만 번) 제한에 걸려 무조건 **시간 초과**가 납니다.

    * *결론:* **"구간의 길이가 고정되어 있는데 $N$이 매우 크다? $\rightarrow$ 무조건 슬라이딩 윈도우!"** 라는 공식을 바로 떠올리셔야 합니다.
* **핵심 목표:** 매번 구간 합을 구하지 않고, 양 끝점의 값만 더하고 빼서 $O(N)$ 속도로 최대 구간 합 구하기.
* **입력 범위 및 시간 복잡도:** $N \le 100,000$이므로 배열을 딱 한 번만 훑고 지나가는 $O(N)$ 시간 복잡도로 해결해야 합니다.
* **주의할 엣지 케이스:** 주어지는 온도가 **모두 음수**일 수 있습니다. (예: -10, -5, -8 ...)

### 2. 🧠 논리적 설계 (주석 뼈대)

작성하신 훌륭한 슬라이딩 윈도우 로직의 정석적인 뼈대입니다.

```markdown
1. 전체 날짜 수 N과 연속 구간 길이 K를 입력받는다.
2. 전체 온도 배열을 입력받는다.
3. 처음 K일 동안의 온도 합을 미리 구하여 '현재 윈도우의 합(tem)'으로 설정한다.
4. 이 초기 합을 '최댓값(max_val)' 변수에 기록해 둔다.
5. K번째 인덱스부터 배열의 끝까지 윈도우를 한 칸씩 우측으로 밀며 순회한다:
   5-1. 윈도우에서 빠져나가는 맨 앞의 값을 빼고, 새로 들어오는 맨 뒤의 값을 더한다.
   5-2. 갱신된 현재 윈도우의 합과 최댓값을 비교하여 최댓값을 갱신한다.
6. 순회가 끝나면 최종 최댓값을 출력한다.
```

### 3. 💻 코드 구현 (Pythonic) 리뷰

기존에 시간 초과가 났던 이유는 아마 `for`문 안에서 매번 `sum(temperatures[i : i+K])`를 호출하셨기 때문일 것입니다. 슬라이싱과 `sum()`은 내부적으로 배열을 도는 $O(K)$
연산이므로 루프 안에서 쓰면 TLE의 주범이 됩니다.

현재 코드는 그 함정을 완벽히 빠져나와 덧셈과 뺄셈 1번씩만으로 합을 갱신하는 정답 로직입니다.

**[아쉬운 점 & 최적화 포인트]**
`results` 리스트를 만들어 모든 윈도우의 합을 계속 `append` 한 뒤 마지막에 `max(results)`를 호출하고 계십니다. 메모리 제한이 넉넉해서 통과는 했지만, 데이터가 1000만 개라면 리스트의
크기가 기하급수적으로 커져 메모리 초과(MLE)가 날 수 있습니다. 모든 기록을 남길 필요 없이, **'가장 큰 값 하나'** 만 실시간으로 변수에 들고 있으면 메모리를 극도로 절약할 수 있습니다.

### 4. 🛡️ 엣지 케이스 점검

* **배열의 모든 값이 음수인 경우:** 초기 최댓값 변수를 습관적으로 `0`으로 설정하면, 정답이 -15인데 0이 출력되는 대참사가 벌어집니다.
* 하지만 사용자님은 초기 구간 합을 먼저 구한 뒤 그것을 리스트에 넣고 시작했기 때문에, 음수 엣지 케이스도 완벽하게 빗겨나가는 견고한 방어력을 보여주셨습니다.

### 5. 🏆 최고 수준 최적화 (Review & Optimization)

리스트(`results`)를 제거하여 공간 복잡도를 $O(N)$에서 $O(1)$로 줄이고, 파이썬 특유의 간결한 언패킹으로 가독성을 높인 최적화 코드입니다.

```python
import sys

input = sys.stdin.readline

N, K = map(int, input().split())
temps = list(map(int, input().split()))

# 1. 초기 윈도우(K개)의 합을 구합니다.
current_sum = sum(temps[:K])

# 2. 엣지 케이스(모두 음수)를 방지하기 위해 초기 윈도우의 합을 최댓값으로 세팅합니다.
max_sum = current_sum

# 3. 윈도우를 한 칸씩 이동시키며 O(1)로 합을 갱신합니다.
# 새로 들어올 인덱스를 i로 두고 범위를 K부터 N-1까지로 잡으면 인덱싱이 훨씬 깔끔해집니다.
for i in range(K, N):
    # 윈도우 갱신: (기존 합) + (새로 들어오는 값) - (빠져나가는 맨 앞의 값)
    current_sum += temps[i] - temps[i - K]

    # 4. 리스트에 저장하지 않고 실시간으로 최댓값만 갱신합니다.
    if current_sum > max_sum:
        max_sum = current_sum

print(max_sum)
```

💡 **최적화 핵심 원리:**

* **공간 복잡도 최적화:** 히스토리를 전부 기억할 필요가 없는(최대/최소만 구하는) 문제에서는 배열 대신 변수 하나를 갱신해 나가는 것이 코딩테스트 메모리 관리의 기본입니다.
* **인덱싱 가독성:** `i - 1`과 `i + (K - 1)` 대신, 반복문을 `K`부터 시작하게 만들어 새로 들어오는 값을 `temps[i]`, 빠지는 값을 `temps[i - K]`로 직관적으로
  매핑했습니다. 인덱스 계산 실수를 줄여주는 패턴입니다.
